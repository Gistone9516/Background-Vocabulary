# 배경노트 (Vock note)

막연한 생각을 가진 사용자가 "설명(표현)하지 않고도" 클릭·재인만으로 의도를 결정화해, 자기 상황에 맞는 배경지식 어휘를 얻는 도구. 제품 핵심 축 = **표현(articulation) 비용 면제** 단일 축.

## 구조

```
v1/   크롬 확장(사이드패널) 구현 — 동결(2026-07-20). 참조·자산 창고
      ├ docs/      기획·설계 정본 9종(기획-구체화, 인터페이스계약(SoT), 결제인증 설계,
      │            구현계획, 배포 가이드, 이관, 사용자판단대기, 비용수익 예측, 평가-비판검토)
      ├ sidetab/   모노레포 코드(core/shared/adapters 포트-어댑터, Workers+Hono, React 사이드패널)
      ├ UI_UX/     디자인 참고 자산
      └ README.md  v1 상세(실행·빌드 방법 포함)
v2/   리빌딩(진행 중) — 웹사이트 + 데스크톱 프로그램(Windows·Linux 호환)
      └ docs/v1-승계원장/   v1 기획·설계 무손실 인벤토리(v2 요구사항 명세의 입력)
```

- 폼팩터: 크롬 확장 → **웹사이트 + 크로스플랫폼 데스크톱 앱**.
- 방법론: 소프트웨어 공학 명세 절차(요구사항 명세 → 아키텍처 설계 → 상세설계 → 구현 → 테스트). 디자인 패턴·계층 분리 기반의 유지보수 가능한 코드, 적층형 코딩 금지.
- v1 기획은 **요약 없이 원문 그대로 승계**(승계원장). 디자인 컨셉은 v1 유지(덜어내고 차분하게), 더 나은 안은 변경 후보로 검토.
- 백엔드 자산(Workers+Hono, D1 엔타이틀먼트, 서버 JWT, Google SSO, 프롬프트 파이프라인)은 클라이언트 폼팩터와 무관하므로 재사용 우선 검토.
