## §1 가정·입력값

이 섹션은 §2~§6의 모든 계산이 딛고 서는 단가·요율·요건과 운영 가정을 한곳에 모은다. 원칙은 두 가지다. ① 단가·요율·법적 요건은 모두 웹 근거의 출처를 단다. ② 출처가 없는 수치(세션당 토큰량, 사용자당 월 세션 수, 캐시 적중률 등)는 `[추정]`으로 표시하고 산출 논리를 같이 적되, 신뢰도를 낮춰 둔다. 본 보고서는 과거 추정치(DeepSeek $0.27/$0.87, CF $5/월 등)를 신뢰 입력으로 쓰지 않으며, 아래 표의 출처 확인값만 입력으로 삼는다.

### 1.4 세션당 토큰량 가정 `[추정]`

위 코드 상한을 토대로, "전형적 1세션"의 토큰량을 추정한다. 출력은 상한(`max_tokens`)을 그대로 쓰지 않고 **실측 전형값(상한의 60~90%)**을 가정한다. 입력은 시스템 프롬프트(고정, 캐시 대상)와 가변 입력(사용자 입력·누적 히스토리·RAG 스니펫)으로 나눈다. 시스템 프롬프트는 길고 반복되므로 입력 토큰의 큰 몫을 차지하며, 이것이 §1.6 캐시 가정과 직결된다.

가정 신뢰도: **L(낮음)** — 실 사용 로그가 없어 프롬프트 길이와 전형 출력 비율을 보수적으로 잡았다. 운영 후 실측으로 교체해야 한다.

> **★ 본 세션 모델은 변동비의 하한이다(미반영 비용 경로 존재).** 실제 코드·UX에는 추가 LLM 호출을 일으키는 경로가 더 있으며, 추가 호출 = 추가 출력 토큰(비용 지배 요소)이다. 아래 표는 이 경로들을 **명시적으로 포함**해 보정했고, 미반영분은 본 절 끝 각주에 목록으로 둔다. 따라서 §2 변동비는 "이 경로들까지 반영한 값"으로 읽되, 여전히 일부 꼬리 경로가 빠져 **체계적으로 과소가 아닌 보수(하한+α) 추정**임을 전제한다.

> **각주 — §1.4 미반영(또는 보수 반영) 비용 항목.** 아래 경로는 본 모델에 일부만 반영되었거나 빠져 있어, 현 변동비는 여전히 **하한+α**다. 운영 로그로 교체 시 상향 가능성이 크다.
> 1. **그룹 추가 생성의 꼬리 분포** — pro `maxTotal`=32를 적극 채우는 헤비 유저는 그룹 추가 생성이 3회를 크게 초과(최대 7~8회)할 수 있다. 본 모델은 평균 3회만 반영.
> 2. **simplify 다회 발생** — "선택지가 어려워요"를 한 세션에서 2회 이상 누르는 경우는 미반영(평균 0.3~0.5회만 반영).
> 3. **스트리밍 중단 후 전체 재생성** — 부분 출력 후 사용자가 처음부터 다시 받는 경우, +8% 균등 가산으로는 과소평가될 수 있다.
> 4. **검색 재탐색(pro append)** — 검색창에서 조건을 바꿔 RAG를 다시 도는 경우의 추가 Tavily 호출·추가 입력 토큰은 §1.6 캐시 미스율에만 일부 흡수되고 토큰 모델엔 미반영.
> 5. **에러 후 사용자 재시작** — 세션 자체를 다시 시작하면 classify부터 재발생하나 별도 세션으로 계산되어 본 세션 모델 바깥.

## §2 비용 모델(고정+변동·10^n)

비용을 **고정비**(사용자 수와 무관한 인프라 월정액)와 **변동비**(세션 수에 비례하는 LLM·검색·캐시 호출)로 나눠 모델링한다. 환율은 보수적으로 **1 USD = 1,450 KRW**(상단 가정, [추정])로 고정한다. 단가는 §1·웹 근거의 출처를 그대로 사용하고, 사용 패턴(세션당 토큰·호출 수, 월 세션 수)은 [추정]으로 명시한다.

### 2.2 세션당 변동비 — 호출·토큰 가정과 계산

**호출 패턴**(제품 정의): classify 1회 + nextBranch(무료 3, 유료 8) + recommend 스트리밍 1회(무료 8어휘, 유료 32어휘) + detail 온디맨드(무료 3, 유료 실사용 6) + summarize(유료만 1).

**캐시 단가 모델**: 반복되는 시스템 프롬프트는 DeepSeek 프롬프트 캐시 대상이다. 시스템 프롬프트 토큰의 **캐시 적중률 60%**를 가정한다([추정], 신뢰도 M — 동일 시스템 프롬프트를 세션 내 반복 호출하므로 보수적 하한). 가변 입력은 항상 캐시 미스로 계산한다.

호출별 토큰 가정([추정], 프롬프트 구조 기반):

| 호출 | 시스템(캐시대상) | 가변입력 | 출력 |
|---|---|---|---|
| classify | 900 | 200 | 350 |
| nextBranch | 700 | 400 | 300 |
| recommend(무료 8) | 1,200 | 900 | 2,200 |
| recommend(유료 32) | 1,200 | 1,100 | 7,000 |
| detail | 800 | 300 | 600~700 |
| summarize(유료) | 1,000 | 1,500 | 900 |

**호출당 비용 계산식**(캐시적중률 h, 시스템 S, 가변입력 I, 출력 O):
```
입력비 = [(S·(1−h) + I) / 1e6]·단가_in  +  [(S·h) / 1e6]·단가_hit
출력비 = (O / 1e6)·단가_out
```
단가(USD/MTok, 출처: [DeepSeek Pricing](https://api-docs.deepseek.com/quick_start/pricing)):

- pro: in $0.435 / hit $0.003625 / out $0.87

| nextBranch | 3 | $0.000180 | $0.000541 |

| detail | 3 | $0.000256 | $0.000768 |
| **LLM 합계** | | | **$0.002299** |

- RAG(Tavily): 세션당 검색 시도 1회, 도메인 캐시 적중률 70%([추정]) → 실호출 0.30회 × $0.008 = **$0.002400**
- Upstash: 세션당 15명령([추정], 카운터·캐시 read/write) × $0.000002 = **$0.000030**
- **무료 세션 총 변동비 = $0.004729 ≈ 6.86원**

**유료(pro) 세션** — 검증된 산출:

### 2.3 사용자 규모 10^n 월 비용 표

**월 세션 수 가정**([추정], 신뢰도 M):
- 무료 사용자: **12세션/월**(주 3회 × 4주, 주 7회 한도 내)
- 유료 사용자: **30세션/월**(일 1회 수준)
- **유료 비중 5%**(본 절은 5% 고정으로 절대 비용 산출; 전환율 민감도는 §3·§4)

**월 비용 계산식**:
```
무료 변동비 = (N·0.95) × 12세션 × $0.004729
유료 변동비 = (N·0.05) × 30세션 × $0.021141
월 총비용  = 무료 변동비 + 유료 변동비 + $5(CF 고정)
```

| N(명) | 무료/유료 | 무료 변동비 | 유료 변동비 | +CF 고정 | 월 총비용(USD) | 월 총비용(KRW) |
|---|---|---|---|---|---|---|
| **10** | 10 / 0 | $0.54 | $0.32 | $5 | **$5.86** | **약 8,500원** |
| **100** | 95 / 5 | $5.39 | $3.17 | $5 | **$13.56** | **약 19,700원** |
| **1,000** | 950 / 50 | $53.91 | $31.71 | $5 | **$90.62** | **약 131,400원** |
| **10,000** | 9,500 / 500 | $539.07 | $317.11 | $5 | **$861.18** | **약 124.9만원** |
| **100,000** | 95,000 / 5,000 | $5,390.70 | $3,171.13 | $5 | **$8,566.83** | **약 1,242만원** |

검산(N=1,000): 무료 950 × 12 × $0.004729 = $53.91, 유료 50 × 30 × $0.021141 = $31.71, 합 $85.62 + $5 = **$90.62**. ✔

**사용자 1인당 월 원가**(N=100,000 기준): $8,566.83 ÷ 100,000 = **$0.0857 ≈ 124원/인·월**. 유료 1인당 원가만 보면 30세션 × $0.021141 = $0.634 ≈ **920원/유료·월**으로, 가격 3,500원 대비 원가율 약 26%다(마진 상세는 §4).

### 2.4 무료 한도 소진 시점(고정→변동 전환)

규모가 커지면 Tavily·Upstash가 무료 한도를 넘어 변동비로 전환되는데, **위 표는 이미 종량 단가로 계산**했으므로(무료 한도를 비용 0으로 깔지 않음) 보수적 상단이다. 한도 소진 시점만 별도로 짚는다:

| N | Tavily 호출/월 (무료 1,000) | Upstash 명령/월 (무료 50만) | CF 요청/월 (포함 1,000만) |
|---|---|---|---|
| 10 | 39 | 2,085 | 1,167 |
| 100 | **387** | 20,850 | 11,670 |
| 1,000 | 3,870 ⚠ | 208,500 | 116,700 |
| 10,000 | 38,700 ⚠ | **2,085,000** ⚠ | 1,167,000 |
| 100,000 | 387,000 ⚠ | 20,850,000 ⚠ | 11,670,000 ⚠ |

- **Tavily**: N≈260명에서 월 1,000크레딧 소진(무료 종료). 이후 종량 $0.008/회 또는 구독 플랜($0.005~$0.0075/크레딧)으로 단가 인하 가능. 위 표는 종량 $0.008로 계산.
- **Upstash**: N≈2,400명에서 50만 명령 소진. 이후 $0.20/10만 명령. N=100,000에서 약 2,085만 명령 = **$41.70/월**(전체 비용의 0.5% 미만, 미미).
- **Cloudflare(요청 수)**: N=100,000에서도 월 요청 약 1,167만으로 포함 1,000만을 약간 초과 → 초과 167만 × $0.30/백만 = **$0.50/월**.

**Cloudflare CPU-ms 점검(적대 검증 반영)**: LLM 스트리밍의 순수 I/O 대기는 과금 대상이 아니지만, 스트림 파싱·SSE 변환·JSON 재조립은 실제 CPU를 쓴다. 세션당 워커 CPU 사용을 **약 50 CPU-ms**(요청 라우팅·검증·SSE 파싱·캐시 read/write 합산, [추정], 신뢰도 L — 미실측)로 잡으면:

```
월 CPU-ms = (무료 N·0.95 × 12세션 + 유료 N·0.05 × 30세션) × 50 CPU-ms
```

| N | 월 세션 수(무료+유료) | 월 CPU-ms(50ms/세션) | 포함 3,000만 대비 |
|---|---|---|---|
| 10,000 | 114,000 + 15,000 = 129,000 | 약 645만 | 여유(약 21%) |
| 100,000 | 1,140,000 + 150,000 = 1,290,000 | 약 6,450만 ⚠ | **초과(약 2.15배)** |

N=100,000에서 추정상 포함 한도를 넘을 수 있다. 초과분 약 3,450만 CPU-ms × $0.02/백만 = **약 $0.69/월**. 세션당 CPU-ms를 두 배(100ms)로 잡아도 초과 과금은 약 $2.6/월 수준이라 전체 비용($8,566)의 0.1% 미만이다. 다만 이 50ms는 미실측 추정이므로 **CPU-ms 초과 가능성을 0으로 단정하지 않고, 추정상 여유는 있으나 배포 후 워커 CPU-ms 실측이 필요한 항목으로 남긴다**(특히 스트리밍 호출 다회·대형 JSON 재조립 구간).

**결론**: 전 구간에서 비용의 90% 이상이 **DeepSeek LLM 변동비**다. 인프라(CF 요청·CPU·Tavily·Upstash)는 10만 명에서도 합산 $43~$45/월 수준(요청 초과 $0.50 + CPU 추정 $0.69 + Upstash $41.70)으로 비용 구조의 곁가지다. 따라서 단위경제성은 **(1) 유료 세션의 출력 토큰량, (2) 캐시 적중률, (3) 유료 비중** 세 변수가 지배한다(민감도는 §4).

> 가정 일람(모두 [추정], 출처는 단가만 적용): 환율 1,450 KRW/USD · 캐시 적중률 60% · RAG 도메인 캐시 적중 70% · 무료 12세션/월 · 유료 30세션/월 · 유료 비중 5% · 세션당 토큰량(표 2.2) · 세션당 워커 CPU 약 50ms(신뢰도 L, 미실측). 단가는 전부 웹 근거(신뢰도 H). 가정값이 바뀌면 비용은 선형 비례하므로 §4 민감도 표로 대체 시나리오를 제공한다.

## §3 수익 모델(전환 시나리오)

본 절은 매출 측면만 다룬다. 비용(고정비·변동비)은 §2에서 산출했고, 둘을 맞붙인 손익분기·마진은 §4에서 다룬다. 여기서는 **유료(pro) 가격 × 전환율(low/mid/high) × 사용자 규모(10^n)** 격자로 월 매출만 계산한다.

### 3.1 매출 모델의 구조와 가정

배경노트의 수익원은 단일하다. 무료(flash) 사용자는 매출 0원이고, 유료(pro) 구독자만 **월 정액**을 낸다. 광고·인앱 추가결제·기업 라이선스는 현 기획 범위 밖이라 0으로 둔다. 따라서 월 매출은 다음 한 줄로 결정된다.

```
월 매출(총) = MAU × 전환율 × pro 월 가격
월 매출(순) = MAU × 전환율 × pro 월 가격 × (1 − PG 유효수수료율)
```

여기서 MAU는 월 활성 사용자, 전환율은 그중 유료로 전환한 비율이다. 매출을 좌우하는 입력값과 그 근거는 다음 표와 같다.

| 입력값 | 채택값 | 출처 / 논리 | 신뢰도 |
|---|---|---|---|
| pro 월 가격 | 3,500원(기준) / 2,900·4,900원(대안) | 기획 잠정가 3,500원. 가격 민감도를 보려 3개 점으로 격자화 | 기준값=기획 확정, 대안=[추정] M |
| 전환율 low / mid / high | 1% / 3% / 6% | 프리미엄 SaaS·확장프로그램 유료전환은 통상 1~5%대. 본 서비스는 무료 한도(주 7회)가 명확해 전환 압력이 있으나 비전문가 타깃이라 지불의사 약함 → low~mid를 현실 기대, high는 낙관 상한 | [추정] M |
| PG 유효수수료율 | 3.74% | 토스페이먼츠 일반 가맹점 3.4%(VAT 별도)에 부가세 10%를 수수료에 가산하면 실효 약 3.74%. 출처: 토스페이먼츠 PG 수수료 <https://www.tosspayments.com/about/fee> | H |

| USD/KRW 환율 | 1,450원/USD | 비용은 USD, 매출은 KRW이므로 §4 대조용으로 명시. 매출 계산 자체엔 KRW만 쓰여 환율 영향 없음. 2026-06 근사값 | [추정] M |
| MAU 규모 | 10 / 100 / 1,000 / 10,000 / 100,000명 | 과제 지정 10^n 스케일(최대 10만) | 지정값 |

**가정에 대한 주의 두 가지.**
첫째, **전환율은 본 모델에서 가장 불확실하고 가장 영향이 큰 변수**다. 외부 실측 출처가 없어 [추정]이며, low~high 사이에서 매출이 6배 벌어진다. §4 민감도에서 이 변수를 핵심으로 다룬다.
둘째, 여기서 말하는 전환율은 "MAU 대비 유료 비중"이다. §2의 비용 계산에서 쓰는 flash:pro **트래픽 비율**과는 다른 개념이다. 비용 쪽은 무료 사용자도 호출 비용을 발생시키지만, 매출 쪽은 무료 사용자가 0원이므로, 같은 사용자 풀을 매출은 "유료 비중"으로, 비용은 "전체 호출량"으로 본다. 이 비대칭이 §4 손익분기의 핵심이다.

### 3.4 ARPU(전체 MAU 1인당 월매출)

비용은 §2에서 사용자당 USD로 잡혔으므로, 매출도 "전체 MAU 1인당"으로 환산하면 §4에서 직접 빼서 마진을 본다. 기준가 3,500원(순수취 3,369원) 기준 ARPU는 전환율에만 비례한다(규모 무관).

| 전환율 | ARPU(순, KRW/MAU) | USD 환산(1,450원/USD) |
|---|---|---|
| low (1%) | 34원 | 약 $0.023 |
| mid (3%) | 101원 | 약 $0.070 |
| high (6%) | 202원 | 약 $0.139 |

> 계산: ARPU = 순수취 3,369원 × 전환율. 예) mid = 3,369 × 0.03 ≈ 101원. USD = 101 / 1,450 ≈ $0.070.

이 값은 §2에서 산출한 사용자당 원가(블렌드 기준 약 $0.012/MAU 수준, 단 §2의 최신 규모별 재계산 결과를 정본으로 봄)와 직접 대비된다. **mid 전환(3%)에서 ARPU $0.070 > 사용자당 원가**이면 변동비 단위경제가 흑자라는 신호이고, 여기에 고정비(인프라 월정액·PG 연관리비·법무)를 매출이 덮는 지점이 진짜 손익분기다. 그 임계 전환율 계산은 §4로 넘긴다.

### 3.5 읽는 법(요약)

- **소규모(10~100명):** 어느 가격·전환율이든 월 매출이 1만 원 미만이다. 이 구간은 매출이 아니라 사용자 확보·전환율 검증 구간으로 봐야 한다.
- **1,000명:** mid 전환·3,500원에서 월 순매출 약 10만 원. 인프라 고정비(§2: Workers $5 + Tavily·Upstash 무료 한도 내)는 덮지만, PG 연관리비·법무·개발 공수까지 보면 아직 적자 가능성이 높다.
- **10,000명:** mid·3,500원에서 월 순매출 약 101만 원. 이 구간부터 매출이 변동비를 확실히 상회한다(§2의 1만 명 변동비는 백 달러대 = 십수만 원대). 단위경제가 처음으로 의미 있게 양(+)이 되는 분기점.
- **100,000명:** mid·3,500원에서 월 순매출 약 1,011만 원, high면 약 2,021만 원. 이 규모에서는 가격(2,900 vs 4,900)과 전환율 차이가 절대 금액으로 수백만~수천만 원을 가른다 → §6의 가격·전환 전략이 핵심 레버가 되는 구간.

**핵심 비대칭:** 매출은 전환율과 가격에 선형이지만, 비용은 전체 호출량(무료 포함)에 끌려간다. 따라서 전환율이 낮으면 "비용은 다 받는데 매출은 일부만 받는" 구조가 된다. 이 점이 §4 손익분기와 §6 권고(무료 한도 조정·모델 폴백으로 무료 사용자 원가 억제)의 출발점이다.


## §4 손익·민감도

> 본 절은 §1~§3의 입력값·가정을 그대로 잇는다. 모든 단가는 프롬프트에 첨부된 웹 근거(asof 2026-06-27, confidence H)를 출처로 인용하며, 출처가 없는 가정값에는 **[추정]**과 신뢰도 표기를 붙인다. USD/KRW 환율은 **[추정] 1 USD = 1,380원**(신뢰도 M, 별도 실시간 출처 미확보 — 결제·정산 시점 실환율로 재검증 대상)을 단일 가정으로 쓴다. 계산은 모두 식을 노출해 검증 가능하게 적었다.
>
> **[정정 — 적대 검증 반영]** 직전 판본은 "세션당 검색 시도 무료 2회/유료 3회"로 잡아 §2.2(1회)와 모순됐다. **구현 코드로 확정한다: 세션당 RAG 검색 시도는 1회다.** RAG는 `core/rag/runRag()`가 담당하고, 이 함수는 추천 1사이클(`recommendStream`)당 정확히 한 번 호출되며 그 안에서 캐시 조회 후 미스일 때만 Tavily Basic을 **1회** 호출한다(`deps.search.search(...)` 단일 호출, `depth: "basic"`). 따라서 §2.2와 §4를 **1회로 통일**하고 아래 전 계산을 재산출했다. 또한 Tavily **무료 1,000크레딧/월** 한도 내 구간은 RAG 실비용을 **0원**으로 명시한다(§4.0 단서). 이 통일로 "RAG가 LLM보다 비싸다"·"RAG가 1순위 레버"라는 직전 단정은 **조건부로 약화**했다(아래 주석).

**세션당 변동비(식 노출, flash=무료, pro=유료)**

LLM 비용 = (입력×적중률)/10⁶×캐시히트단가 + (입력×(1−적중률))/10⁶×미스단가 + 출력/10⁶×출력단가

- 무료(flash): (12,000×0.7)/10⁶×0.0028 + (12,000×0.3)/10⁶×0.14 + 6,000/10⁶×0.28 = **$0.00221**
  - 단가 출처: DeepSeek V4 Flash 입력 미스 $0.14, 히트 $0.0028, 출력 $0.28 (api-docs.deepseek.com, H)
- 유료(pro): (30,000×0.7)/10⁶×0.003625 + (30,000×0.3)/10⁶×0.435 + 20,000/10⁶×0.87 = **$0.02139**
  - 단가 출처: DeepSeek V4 Pro 입력 미스 $0.435, 히트 $0.003625, 출력 $0.87 (api-docs.deepseek.com, H)

RAG 비용 = 세션당 검색 시도(**1회, 코드 확정**) × (1−도메인캐시) × $0.008 (Tavily Basic 1크레딧, docs.tavily.com, H)
- 무료: 1 × 0.4 × $0.008 = **$0.00320**
- 유료: 1 × 0.4 × $0.008 = **$0.00320**

> **★ Tavily 무료 한도 단서(반드시 함께 읽을 것).** 위 $0.008은 **종량제(무료 1,000크레딧/월 소진 이후)** 단가다. Tavily는 신용카드 없이 **월 1,000크레딧 무료**(Basic 1회=1크레딧, docs.tavily.com, H). 세션당 청구 크레딧이 0.4(=1회×미스율 0.4)이므로 **월 누적 청구 검색이 1,000크레딧을 넘기기 전에는 RAG 실비용 = 0원**이다. 무료 한도가 덮는 규모[추정 M]: 전원 무료(12세션)면 약 **208명**(=1,000÷(0.4×12)), 전원 유료(40세션)면 약 **62명**(=1,000÷(0.4×40))까지. 즉 초기 소규모 구간에서는 아래 RAG 변동비를 0으로 봐도 무방하며, **"RAG 변동비"가 실제로 발생하는 것은 무료 한도를 넘긴 성장 단계부터**다. 아래 표·BEP는 보수적으로 무료 한도를 무시하고 종량 단가를 곧장 적용한 값이다(상한 추정).

Redis = 세션당 명령 × $0.000002 (Upstash $0.20/10만, upstash.com/pricing/redis, H)
- 무료: 20 × $0.000002 = $0.00004 / 유료: 40 × $0.000002 = $0.00008

| 항목 | 무료(세션) | 유료(세션) |
|---|---|---|
| LLM | $0.00221 | $0.02139 |
| RAG (종량 가정) | $0.00320 | $0.00320 |
| Redis | $0.00004 | $0.00008 |
| **세션 합계** | **$0.00545 ≈ 7.5원** | **$0.02467 ≈ 34.1원** |

**사용자당 월 변동비(종량 가정)**
- 무료: $0.00545 × 12 = $0.0654 ≈ **90원/월**
- 유료: $0.02467 × 40 = $0.9868 ≈ **1,362원/월**

> **변동비 구성 재해석(직전 단정 철회).** 검색 시도를 1회로 통일하면 무료 세션 7.5원 중 **RAG 4.4원(58.7%), LLM 3.0원(40.5%)**로, RAG가 여전히 약간 크지만 직전 판본의 "RAG 74% > LLM"만큼 압도적이지 않다. 게다가 **무료 1,000크레딧 한도 내 초기 구간에서는 RAG 실비용 0원 → LLM이 사실상 유일한 변동비**가 된다. 따라서 "RAG가 손익을 가르는 1순위 레버"라는 직전 단정은 **무료 한도를 넘긴 이후·도메인 캐시가 가정대로 안 들을 때에만 성립하는 조건부 주장**으로 약화한다(§4.4에서 캐시 적중률 가정 의존성으로 다룸). 1순위 레버는 §4.4 결과상 **LLM 단가**다.

### 4.1 유료 사용자 1명의 기여이익(contribution margin)

- 가격 3,500원 (잠정)
- PG 수수료 = 3,500 × 0.034 × 1.10(VAT) = **131원**
- 순수취 = 3,500 − 131 = **3,369원**
- 유료 변동비 = 1,362원
- **월 기여이익 = 3,369 − 1,362 = 2,007원 (순매출 대비 마진율 59.6%)**

유료 1명이 매달 약 2,007원의 마진을 만든다. 이 2,007원이 ① 무료 사용자가 까먹는 변동비와 ② 고정비(CF Workers $5≈6,900원/월, developers.cloudflare.com, H)를 메워야 손익분기에 도달한다.

### 4.2 손익분기 전환율(BEP conversion)

총사용자 N, 유료 전환율 c일 때 월 이익:

이익 = c·N·(유료 기여이익 2,007) − (1−c)·N·(무료 변동비 90) − 고정비 6,900

이익 = 0으로 두고 c에 대해 풀면:

c = (N·90 + 6,900) / (N·(2,007 + 90))

| 총사용자 N | 무료 변동비 총액 | 고정비 | **BEP 전환율 c** | 필요 유료 수 |
|---|---|---|---|---|
| 100 | 9,000원 | 6,900원 | **7.58%** | 약 8명 |
| 1,000 | 90,000원 | 6,900원 | **4.62%** | 약 46명 |
| 10,000 | 900,000원 | 6,900원 | **4.32%** | 약 432명 |
| 100,000 | 9,000,000원 | 6,900원 | **4.30%** | 약 4,295명 |

**해석**
- 규모가 커질수록 고정비(6,900원)가 희석돼 BEP 전환율이 **약 4.30%로 수렴**한다. 이 점근값은 거의 전적으로 "무료 1명 변동비 90 ÷ 유료 1명 기여이익 2,007 ≈ 4.3% 구조비"가 결정한다(고정비 무시 시 c≈90/(2,007+90)=4.29%).
- 즉 **약 4.3% 이상이 유료로 전환하면 흑자**다. 일반적 프리미엄 SaaS 전환율(2~5%) 범위와 겹치므로, **검색 시도를 1회로 통일한 현 구조에서 BEP 4.3%는 도달 가능한 수준**이다(직전 판본의 8% BEP는 검색 시도 과대계상에서 비롯한 오추정이었다).
- 초기 무료 1,000크레딧 한도를 감안하면 소규모 구간 무료 변동비는 90원보다 더 낮아(RAG 0) BEP가 추가로 내려간다. 위 표는 상한(보수) 추정이다.
- 소규모(N=100)에서 BEP가 7.58%로 튀는 건 고정비 6,900원이 분모 대비 크기 때문이다. 초기 단계에서는 고정비 자체가 미미(월 1만원 미만)하므로 절대 적자액이 작아 실질 부담은 낮다.

### 4.3 마진 구조(규모별 절대 손익)

전환율을 시장 통상치 가정 **[추정] low 2% / mid 5% / high 10%** (신뢰도 L — §3과 동일 가정)로 두고 월 순이익을 계산한다.

순이익 = c·N·2,007 − (1−c)·N·90 − 6,900

| N | c=2% | c=5% | c=10% |
|---|---|---|---|
| 100 | −11,706원 | −5,415원 | +5,070원 |
| 1,000 | −54,960원 | +7,950원 | +112,800원 |
| 10,000 | −487,500원 | +141,600원 | +1,190,100원 |
| 100,000 | −4,812,900원 | +1,478,100원 | +11,963,100원 |

> **부호 검증(BEP와 일관성).** 위 표는 "유료수익−무료비용−고정비"를 식 그대로 적용한 값이다. BEP 점근 ≈4.30%이므로 **c=2%는 전 규모 적자, c=5%·c=10%는 (소규모 N=100 제외) 흑자**가 되어야 하고, 표가 이와 일치한다. 검산 예시(N=10,000·c=5%): 500×2,007 − 9,500×90 − 6,900 = 1,003,500 − 855,000 − 6,900 = **+141,600원(흑자)** — c=5%가 BEP(4.32%)를 웃돌므로 흑자. (직전 판본에서 c=5%가 적자였던 것은 BEP가 8%로 과대했기 때문이며, 검색 시도 통일로 해소됐다.)

**규모별 c=10% 흑자 재검산(식 노출)**
- N=1,000: 100×2,007 − 900×90 − 6,900 = 200,700 − 81,000 − 6,900 = **+112,800원**
- N=10,000: 1,000×2,007 − 9,000×90 − 6,900 = 2,007,000 − 810,000 − 6,900 = **+1,190,100원**
- N=100,000: 10,000×2,007 − 90,000×90 − 6,900 = 20,070,000 − 8,100,000 − 6,900 = **+11,963,100원**

핵심: **무료 사용자 한 명당 90원(종량 가정 상한, 무료 한도 내에서는 더 낮음)이 매달 빠져나간다.** 마진의 본질은 "유료 1명이 무료 약 22.3명(2,007÷90)분의 비용을 감당"하는 구조이고, 따라서 BEP는 곧 "유료:무료 ≒ 1:22.3 비율(≈4.3%)"이다. 검색 시도를 1회로 통일하자 무료 1명 변동비가 직전(143원)보다 작아져 BEP가 절반 수준으로 내려갔다.

### 4.4 손익 핵심 변수 민감도

기준 시나리오(유료 기여이익 2,007원, BEP 4.30%)에서 변수 하나씩만 악화시켜 영향을 측정한다.

**(a) LLM 출력 단가 ↑ 2배** (DeepSeek 가격 인상 또는 Pro 모델 비중 상승 가정)
- 유료 LLM $0.02139 → 출력단가 0.87→1.74로 재계산 시 $0.03879, 세션 $0.04207, 월변동비 $1.683 ≈ 2,322원
- 기여이익 = 3,369 − 2,322 = **1,047원** (마진율 59.6% → 31.1%로 급락)
- BEP 전환율 점근 ≈ 90/(1,047+90) = **7.9%** (4.3%→7.9%, 통상 SaaS 상단을 넘는 구간)
- → **단가에 가장 취약(1순위 레버).** DeepSeek 의존이 단일 실패점. Flash가 세계 최저가(출력 $0.28, 차순위 Flash-Lite $1.50의 1/5.4, Haiku 4.5 $5.00의 1/17.9 — 종합 출처 H)인 만큼, 인상 시 동급 대체 부재가 리스크.

**(b) 캐시 적중률 ↓ 0%** (도메인 캐시·프롬프트 캐시 모두 미작동 최악 가정)
- 유료 LLM(캐시0%) = 30,000/10⁶×0.435 + 20,000/10⁶×0.87 = $0.03045
- 여기에 RAG: 도메인 캐시 0%면 검색 시도 1회가 전부 청구(미스율 1.0) → 1×1.0×$0.008 = $0.008
- 세션 = $0.03045 + $0.008 + $0.00008 = $0.03853, 월변동비 ≈ 2,127원
- 기여이익 = 3,369 − 2,127 = **1,242원** (마진율 36.9%), BEP 점근 ≈ 90/(1,242+90) = **6.8%**
- → 입력 캐시·RAG 도메인 캐시 모두 빠지는 최악에서도 흑자 구조는 유지되나 마진이 절반 가까이 줄어든다. **RAG가 손익에 영향을 주는 통로는 바로 이 "도메인 캐시 미작동" 경로**이며, 이것이 §4.0에서 "RAG는 조건부 레버"라 한 조건이다. 도메인 캐시 적중률 모니터링이 실효 점검 항목.

**(c) 유료 헤비유저(세션 ↑)**
- 유료 변동비는 세션당 34.1원으로 선형 증가. **적자전환 세션수 = 순수취 3,369 ÷ 34.1 ≈ 99세션/월**
- 월 100세션: 변동비 3,410원 → 기여이익 **−41원(적자)** / 월 200세션: −3,451원
- → 상위 헤비유저가 마진을 갉는 구조. detail "유료 무제한"이 비용 꼬리. **유료에도 소프트 상한(예: 월 99세션 부근에서 속도·범위 완화)** 필요. (검색 시도 통일로 적자 전환 임계가 79→99세션으로 다소 완화됐다.)

**(d) 환율 ↑ (1,380 → 1,500원)**
- 유료 월변동비 $0.9868 × 1,500 = 1,480원, 기여이익 = 3,369 − 1,480 = **1,889원** (마진율 56.1%)
- → 환율 +8.7%가 마진 −3.5%p. 민감하지만 치명적이진 않음. 원가가 전액 USD이므로 원화 약세 시 마진 압박은 상시 주시.

**(e) 전환율 ↓** — 4.2·4.3에서 정량화: BEP(≈4.3%)를 못 넘기면 규모가 커질수록 적자 폭이 선형 확대(무료 변동비 누적). 전환율은 매출(§3)과 손익 양쪽을 동시에 흔드는 최대 변수.

### 4.5 요약 판단

- **단위경제는 건전**: 유료 1명 월 기여이익 2,007원(마진율 59.6%)으로, 가격 3,500원 대비 비용 구조에 여유가 있다. (검색 시도를 코드 확정값 1회로 통일하며 직전 추정 1,654원보다 상향됐다.)
- **BEP 전환율 ≈ 4.3%는 통상 SaaS(2~5%) 범위 안**이다. 직전 판본의 8% BEP는 검색 시도(2~3회) 과대계상이 원인이었고, 1회 통일로 도달 가능 영역으로 내려왔다. 무료 1,000크레딧 한도까지는 RAG=0이라 초기 BEP는 이보다 더 낮다.
- **최대 리스크 2개**: ① DeepSeek 단가 인상(대체 부재, 마진 59.6%→31.1%, BEP 4.3%→7.9%) ② 전환율이 BEP 미달.
- **RAG는 조건부 2차 레버**: 변동비 비중은 LLM과 비슷하고 무료 한도 내에서는 0이지만, **도메인 캐시가 가정(60%)대로 안 들으면 마진을 갉는다**(§4.4-b). 1순위 레버는 LLM 단가다.
- **권고 방향**(상세는 §6): 도메인 캐시 적중률을 가정대로 유지·모니터해 RAG 변동비 통로를 닫는다 / 유료 detail 소프트 상한(약 99세션)으로 헤비유저 꼬리 차단 / DeepSeek 단가 헤지로 Flash-Lite 폴백 경로 확보 / 원화 약세·단가 인상 대비 가격은 4,000~4,500원 인상 여지 점검(BEP를 추가로 낮춰 안전마진 확보).

### 5.2 통신판매업 신고

**의무 여부.** 전자상거래법 제12조에 따라 통신판매업 신고 의무가 있다. 다만 **면제 기준**이 있다: 직전 6개월 거래횟수 20회 미만 **그리고** 거래금액 1,200만 원 미만인 **개인**은 면제된다. **법인은 면제가 없으며**, 기준을 초과한 개인도 반드시 신고해야 한다(토스페이먼츠, sales-registration).

배경노트는 월 구독 SaaS이므로 가입자가 누적되면 거래횟수가 빠르게 20회를 넘긴다. 월 3,500원 기준 거래금액 1,200만 원은 **누적 약 3,429건**의 결제에 해당하므로, 실무상 **거래횟수(6개월 20회) 기준이 먼저 걸린다**. 즉 유료 가입자가 한 자릿수만 생겨도 면제 구간을 벗어나므로, **출시와 동시에 신고하는 것을 기본 전제**로 삼는다.

**비용.** 신고 수수료 자체는 0원이다(토스페이먼츠). 단 **등록면허세**가 부과되며, 이는 1회성이 아니라 **매년 1월 1일 기준 반복 납부**다. 인구 50만 이상(서울 등) 40,500원, 50만 미만 22,500원, 군 지역 12,000원이다(tokkiplace, 등록면허세).

**절차·소요.** 정부24 온라인 신청 후 1~3 영업일 내 처리되며, 위택스에서 등록면허세를 납부하면 완료된다. 준비 서류는 ① 사업자등록증 사본, ② **구매안전서비스 이용확인증**(PG사가 발급)이다(ZUZU 가이드).

**해야 할 일.**
1. 사업자등록(개인 또는 법인) 완료.
2. PG사(토스페이먼츠 등) 가입 후 구매안전서비스 이용확인증 발급.
3. 정부24에서 통신판매업 신고 → 위택스 등록면허세 납부.
4. 매년 1월 등록면허세 반복 납부 일정 등록.

### 5.3 결제 수수료(PG)

**플랫폼 수수료 0%.** 크롬 익스텐션 구독은 앱스토어/구글플레이의 인앱결제와 달리 **플랫폼(구글) 수수료가 없다**. 2021년 2월 Chrome Web Store Payments 종료 이후 구글은 구독 결제에 수수료를 징수하지 않으며, 외부 PG사를 자유롭게 연결할 수 있다(ExtensionPay 문서). 따라서 **결제 수수료 부담은 PG사 몫만 남는다**.

**PG사 선택.** **Stripe는 한국 법인 계정 개설이 불가**하므로(2026.01 기준, 인블로그), 토스페이먼츠·나이스페이·KG이니시스 등 **국내 PG사**가 현실적 선택지다.

**수수료율.** 토스페이먼츠 기준 **신규(일반) 가맹점 3.4% + VAT 별도**다. 매출 규모가 커지면 영세·중소 우대요율(영세 0.63% / 중소1 1.15% / 중소2 1.40% / 중소3 1.74%)이 적용되지만, **초기 배경노트는 매출 규모가 영세 기준 미달일 가능성이 높아 3.4%를 기본 적용**한다(토스페이먼츠 PG 수수료). 연 관리비 110,000원이 별도다.

**단가가 낮은 구독에서 수수료의 영향.** 월 3,500원 구독 1건당 수수료를 계산하면:

```
수수료 = 3,500원 × 3.4% × (1 + VAT 0.10)
       = 3,500 × 0.034 × 1.10
       = 3,500 × 0.0374
       = 130.9원  (약 131원)
실수령 = 3,500 - 131 = 3,369원  (결제액의 약 96.3%)
```

출처(토스페이먼츠 PG 수수료)는 VAT 별도 3.4% 기준으로 "약 119원"을 예시하나(3,500 × 0.034 = 119원), VAT 10%를 포함하면 약 131원이다. 보고서 §4 마진 계산에는 **VAT 포함 131원/건(결제액의 약 3.74%)**을 보수적으로 쓴다. 가상계좌 결제는 건당 400원 정액이라 3,500원 구독에서는 카드 결제(3.74%)보다 불리하므로(400/3,500 ≈ 11.4%), **카드·간편결제 유도**가 마진에 유리하다.

연 관리비 110,000원의 단가 환산은 가입자 수에 반비례한다. 예: 유료 100명이면 1인당 연 1,100원(월 약 92원), 유료 1,000명이면 1인당 연 110원(월 약 9원)으로 희석된다.

**해야 할 일.**
1. 국내 PG사 계약(토스페이먼츠 권장: 구매안전서비스 확인증 발급이 통신판매업 신고에도 필요).
2. §4 마진 계산에 **건당 131원(3.74%) + 연 관리비 110,000원**을 변동비로 반영.
3. 가능하면 카드·간편결제를 기본 결제수단으로 노출(가상계좌 400원 정액 회피).

### 5.4 개인정보 처리

**의무 여부.** 개인정보보호법은 **1인 개발자·소규모 스타트업도 예외 없이** 적용한다. 배경노트가 확장 프로그램이고 서버에 본문을 저장하지 않더라도, **이메일·결제정보를 수집하는 순간 개인정보처리자에 해당**한다. 또한 Upstash Redis에 저장되는 **사용자 카운터(주 7회 한도, 세션 카운트 등)도 위탁 처리에 해당할 수 있다**(CELA 작성지침; 넥스트유니콘).

**해야 할 두 가지.**
1. **개인정보 처리방침 작성·공개**: 서비스 내 접근 가능한 위치(확장 설정·옵션 페이지, 가입 영역 등)에 공개해야 한다. 2025년 4월 작성지침이 개정되었으므로 최신 지침을 따른다(CELA, 2025.04).
2. **위탁(수탁자) 내역 고지(개보법 제26조)**: 처리방침에 **Cloudflare(Workers 런타임), Upstash(Redis 카운터/캐시), Tavily(검색), DeepSeek(LLM), PG사** 등 데이터가 흐르는 수탁자를 명시해야 한다. 특히 사용자 입력·세션 데이터가 **해외(DeepSeek/Tavily 등)로 이전**되면 **국외 이전 고지**가 추가로 필요할 수 있다 [추정, 신뢰도 M] — 국외 이전 동의·고지 요건은 별도 법률 확인 권장.

**제재 수위(리스크).** 처리방침 미공개 등 경미 위반은 **과태료 최대 3,000만 원**, 안전조치 미비·유출 시 **과징금 최대 전년도 매출액 3% 또는 20억 원 중 높은 쪽**이다(넥스트유니콘, 신뢰도 M — 출처에서 구체 수치 재확인 권장). 초기 매출이 작아도 처리방침 미공개 자체가 과태료 대상이므로, **출시 전 처리방침 게시는 하드 요건**이다.

**비용.** 자체 작성 시 0원이다. 변호사·전문 템플릿 의뢰 시 [추정] 10~50만 원 수준이나 출처 없음(신뢰도 L). 초기에는 공개 템플릿 기반 자체 작성으로 충분하다고 판단한다.

**해야 할 일.**
1. 출시 전 개인정보 처리방침 작성·게시(확장 옵션 페이지 + 웹사이트).
2. 수탁자 5종(Cloudflare/Upstash/Tavily/DeepSeek/PG) 위탁 내역 명시.
3. 국외 이전 고지 필요 여부 법률 확인(DeepSeek·Tavily 해외 전송).
4. 최소 수집 원칙 적용(이메일·결제 식별자 외 불필요 데이터 미수집 → 컴플라이언스 부담 축소).

### 5.5 AI 생성 결과물 고지(AI기본법)

**의무 여부.** AI기본법은 **2026년 1월 22일 시행**되었고, **모든 생성형 AI 사업자(규모 무관)**에 적용된다. 배경노트는 DeepSeek로 어휘·해설을 생성하므로 생성형 AI 서비스에 해당한다(peekaboolabs; 정책브리핑).

면제 요건(서비스명·화면 표시로 AI 사용이 명백한 경우, 내부 업무용)이 있으나, **'배경노트'는 일반 이용자 대상이라 면제 해당 여부가 불분명**하므로 **명시적 고지를 권장**한다.

**해야 할 일(고지 방법).**
1. **사전 고지**: AI 기반 운용 사실을 이용약관·화면·메뉴에 표시.
2. **결과물 표시**: 어휘 카드·해설 등 각 생성 결과물이 AI로 생성됐음을 명시(예: 카드 하단 "AI 생성" 라벨).

**비용·리스크.** 문구 삽입만으로 충족되어 비용은 0원이다. 위반 시 과태료 3,000만 원 이하지만, **시행 초기 최소 1년(~2027.01)은 계도기간으로 과태료 미부과** 예정이다(정책브리핑). 따라서 **실질 리스크는 2027년 이후**이나, 문구 삽입 비용이 0원이므로 **지금 바로 반영**하는 것이 합리적이다.

### 5.6 RAG 추론 단계 저작권

**현황.** 현행 한국 저작권법에는 TDM/RAG에 대한 명시 면제 규정이 없고, **공정이용(제35조의5) 4요소**로 개별 판단한다: ① 이용 목적·성격, ② 저작물 종류·용도, ③ 이용 비중, ④ 시장·가치에 미치는 영향(법률신문). 2026년 2월 발간된 공정이용 안내서는 **학습 단계 중심**이고 **추론(출력) 단계 가이드는 아직 없다**(정책브리핑, 신뢰도 M).

**배경노트 구조의 책임 분리.** Tavily가 반환한 요약을 DeepSeek가 재서술해 어휘 카드를 만드는 구조에서, **학습 데이터 침해는 Tavily 책임**, **출력 침해는 사업자(배경노트) 책임**으로 분리된다. 배경노트는 ① 단어 수준의 사실 정보 발췌, ② 분야 배경지식 **교육 목적**, ③ 비전문가 대상이라 **공정이용 인정 가능성이 있으나 미확정**이다(법률신문).

**리스크 규모.** 현재 진행 중인 AI 저작권 소송(2025.01 지상파 3사 대 네이버)은 **대형 플랫폼 대상**이며, 배경노트 규모에서 동일 리스크는 낮다(오마이뉴스). 다만 그 판결이 향후 RAG 공정이용 기준을 형성할 수 있으므로 **모니터링이 필요**하다.

**예방조치(해야 할 일).**

### 6.2 R1 — 무료 풀 원가: 가격이 아니라 "소진율"이 레버

§2의 규모별 비용표에서 저전환(1~2%) 구간 총원가의 압도적 비중은 **무료 사용자의 LLM 원가**다. §2 가정대로 무료 사용자가 주 7회 한도를 100% 소진하면, 무료 풀 원가는 사용자 수에 정비례해 깔리는 반면 매출은 전환된 소수에서만 나온다. 따라서:

- **가격 레버(예: 3,000 → 3,500원, +17%)는 BEP를 거의 못 낮춘다.** §4 기준 BEP 전환율은 약 2.5% → 2.1%로 0.4%p만 이동. 이유는 분모(무료 풀 원가)가 가격과 무관하게 그대로이기 때문.
- **소진율 레버는 BEP를 직접 끌어내린다.** 평균 소진율이 100% 가정에서 실측 50%로 내려가면 무료 LLM 원가가 절반이 되어 BEP 전환율이 약 1%대까지 하락(§4 민감도). **즉 "무료 한도 1회 조정"이 "가격 17% 인상"보다 손익에 크게 작용한다.**

근거(단가): DeepSeek V4 Flash 출력 $0.28/MTok, 입력(캐시 미스) $0.14/MTok, 캐시 히트 $0.0028/MTok(약 98% 할인) — DeepSeek API Docs(`https://api-docs.deepseek.com/quick_start/pricing`, asof 2026-06-27). 무료 풀은 flash 전용이므로 무료 원가는 이 단가에 세션당 토큰·세션 수를 곱한 값(§1·§2 계산식 참조).

**권고 R1**
1. **무료 한도를 "주 7회"로 고정하지 말고 베타에서 소진율을 먼저 측정한다.** 평균 소진율이 단일 최중요 KPI다. 측정 전에는 §2의 100% 소진(최악) 가정을 의사결정 기준으로 둔다(보수적).
2. **소진율이 높게 나오면 한도를 주 5회 등으로 조이는 카드를 우선 검토한다.** 단, 너무 빡빡하면 가치 미체감으로 전환 자체가 죽으므로(R2와 상충) "조이되 첫 가치 경험은 보장"하는 균형점을 A/B로 찾는다.
3. **무료 호출 전반의 `max_tokens` 상한을 낮게 유지**(출력 토큰이 비용을 지배 — 출력이 입력의 2배 단가)하여 한 세션당 최악 출력량을 캡한다.

### 6.3 R2 — 전환율: BEP를 넘기는 유일한 흑자 경로

§3·§4에서 보듯 무료 풀 원가가 깔린 구조에서 **유료 전환 1명은 순익을 거의 그대로 더한다**(추가 변동비는 pro 원가뿐이고 매출이 그 위에 얹힘). 따라서 BEP만 넘기면 마진은 가파르게 개선된다(§4의 전환율 1→8% 곡선). 문제는 BEP 자체를 넘느냐다.

- 일반적인 freemium 전환율 벤치마크는 통상 한 자릿수 초반(대략 2~5%)으로 거론된다 — [추정], 신뢰도 낮음(제품·시장·획득 채널에 따라 편차 큼, 본 보고 범위 밖이라 별도 출처 미확보). 이 가정대로면 BEP(§4: 약 2.1~2.5%)는 **달성 가능권의 하단**에 걸친다. 즉 **여유가 크지 않다.**

**권고 R2**
1. **목표 전환율을 BEP+안전마진으로 명시한다.** 예: BEP 2.1%라면 운영 목표를 3% 이상으로 잡아 단가·소진율 변동(R1·R3·R4)의 완충을 확보.
2. **전환은 "한도 소진 순간"의 경험 설계가 핵심.** 무료에서 pro의 차등(추천 어휘 8 → 32개, 턴 3 → 8, detail 무제한, summarize)을 한도 도달 시점에 자연스럽게 보여 전환을 유도. 이는 R1의 "조이기"와 충돌하므로, **한도를 조이되 가치 절정에서 막는** 설계로 둘을 양립시킨다.
3. **전환율을 가격보다 우선하는 레버로 다룬다.** 가격은 BEP를 거의 못 낮추지만(6.2), 전환율 1%p 개선은 §4 표에서 순익을 크게 움직인다.

**권고 R3**
1. **DeepSeek를 1차로 유지하되, 어댑터 계층(프로바이더 추상화)을 둬 폴백을 코드 한 곳에서 전환**할 수 있게 한다. 폴백 후보는 출력 단가가 가장 가까운 **Gemini 3.1 Flash-Lite(5.4x)**를 1순위로.

3. **단가는 분기 단위로 재검증**(메모리 단정 금지 — 본 프로젝트의 과거 환각 사고 교훈). 구 `deepseek-chat/reasoner`는 2026-07-24 deprecate 예정이므로(DeepSeek Docs) **V4 계열 모델 ID 고정**을 배포 전 확인.

### 6.5 R4 — 캐시 적중률: 변동비의 숨은 절반

§1·§2의 비용 계산은 **프롬프트 캐시 적중**과 **RAG 도메인 캐시 적중**을 전제로 한다. 이 두 적중률이 떨어지면 표의 변동비가 함께 상승한다.

- **프롬프트 캐시(입력 토큰):** DeepSeek V4 Flash 캐시 히트 $0.0028/MTok vs 미스 $0.14/MTok — 히트가 미스의 약 1/50(약 98% 절감). 반복 시스템 프롬프트(classify·nextBranch·recommend 등 고정 지시문)가 큰 비중이라 적중률이 입력 비용을 좌우한다(DeepSeek Docs).
- **RAG 도메인 캐시(검색):** Tavily Basic Search 1회 = 1크레딧 = $0.008, 무료 한도 월 1,000크레딧(Tavily Docs). 도메인 캐시 적중 시 검색 호출 자체가 생략 → 검색 변동비 0. 적중률이 낮으면 호출이 늘어 무료 1,000크레딧을 빠르게 소진하고 종량 $0.008/건이 붙는다.

**권고 R4**
1. **두 캐시 적중률을 운영 KPI로 상시 계측**한다. 입력 비용·검색 비용은 적중률에 거의 선형으로 반응하므로 비용 모니터링의 1차 지표.
2. **시스템 프롬프트를 안정적으로 고정**(빈번한 문구 변경은 캐시 무효화 → 미스 폭증)하고, 사용자 가변부는 프롬프트 뒤쪽에 배치해 접두 캐시 적중을 극대화.
3. **RAG 도메인 캐시 TTL·키 설계를 보수적으로** 잡아 영어 쿼리 재사용률을 높인다. Tavily 무료 1,000크레딧 소진 추이를 사용자 규모(§2)와 함께 추적해 종량 진입 시점을 사전 예측.

### 6.6 R5 — 결제 수수료: 소액 구독의 숙명

월 3,500원(잠정) 구독은 객단가가 낮아 **수수료 비율 민감도가 크다.**

- **카드/간편결제(토스페이먼츠 일반 가맹점 3.4%, VAT 별도):** 3,500원 기준 수수료 약 119원(3.4%) + VAT. 영세·중소 우대요율 적용 시 0.63~1.74%로 크게 하락하나, 초기 매출 규모는 우대 기준 미달 가능성이 높아 **일반 3.4% 적용을 보수 가정**(토스페이먼츠 PG 수수료, `https://www.tosspayments.com/about/fee`).
- **가상계좌:** 건당 400원 고정 — 3,500원 대비 11.4%로, 월 구독엔 카드 대비 불리(같은 출처).
- **플랫폼 수수료 0%:** 크롬 웹스토어 구독엔 구글 수수료가 없다(2021-02 Chrome Web Store Payments 종료). 앱/구글플레이 인앱 30%를 회피하는 구조적 이점(ExtensionPay 자료, asof 2021~현재). 즉 **부담은 국내 PG 수수료뿐.**
- **Stripe 한국 법인 불가(2026-01 기준)** → 국내 PG(토스페이먼츠·나이스페이 등)가 현실적 선택지(인블로그, asof 2026-01).

**권고 R5**

2. **가상계좌(건당 400원)를 월 구독 기본 수단으로 노출하지 않는다.** 카드/간편결제를 기본, 가상계좌는 연간권 등 고액 결제에 한정.
3. **우대요율 진입을 적극 신청**한다(매출 성장 시 영세/중소 구간 재산정). VAT 10%·연 관리비를 §5 반복비에 반드시 포함해 마진표를 갱신.

### 6.7 R6 — 제도 리스크: 비용은 작아도 누락이 치명적

직접 현금 비용은 작지만(§5), **누락 시 과태료·서비스 중단**으로 번질 수 있는 영역이다. "돈"이 아니라 "체크리스트 이행"이 리스크 관리의 본질.

| 항목 | 비용 성격 | 핵심 의무 | 출처 |
|---|---|---|---|
| AI 기본법 고지(2026-01-22 시행) | 문구 추가, 사실상 0원 | 생성형 AI 사용 사전 고지 + 결과물 AI 생성 표시 | 정책브리핑·peekaboolabs |

**제도 리스크의 완충재 — 계도기간:** AI 기본법 고지 위반 과태료(3,000만원 이하)는 **시행 초기 최소 1년(~2027-01)이 계도기간으로 미부과 예정**(정책브리핑). 따라서 **실질 금전 리스크는 2027년 이후**다. 단, "지금 안 해도 된다"가 아니라 "지금 붙이면 거의 0원이고 나중에 강제된다"이므로 런칭 시점에 처리하는 것이 합리적.

**권고 R6**
1. **런칭 전 체크리스트로 일괄 처리한다**(돈보다 누락 방지가 목적): ① 약관·UI에 AI 고지 문구 + 생성 결과 라벨링 ② 통신판매업 신고(유료 확정이므로 필수) + 신고번호 푸터 노출 ③ 개인정보 처리방침 공개 + Cloudflare·Upstash **위탁 명시** ④ 출처 URL 병기.
2. **고영향 AI 해당 여부를 확정한다.** 일반 어휘 추천은 비해당 가능성이 높으나, **의료·법률 등 고위험 도메인을 콘텐츠에 포함하면 판단이 달라질 수 있다**(§5). 포함 시 설명요구권·문서보관 등 추가 의무와 법무 검토 비용이 별도 발생하므로 **도메인 포함 정책을 의식적으로 결정**.
3. **저작권은 "원문 대체 가능성"을 낮게 유지**한다. 단어 수준 발췌·사실 정보·교육 목적·출처 표시는 공정이용 인정 가능성을 높이는 예방 조치(미확정이므로 보수적으로). 학습 단계 침해는 모델·검색 공급사 책임 영역, 출력 단계는 사업자 책임으로 분리됨을 유념.
4. **등록면허세는 반복비(연 단위)**임을 §5 반복비표에 반영(서울 기준 약 4.05만원/년).

### 6.8 R7 — 헤비 유저로 인한 pro 마진 잠식

pro는 detail **무제한**·summarize·추천 32어휘·턴 8 등 호출량이 큰 기능을 연다. 평균 사용자 가정으로 짠 §1·§2의 pro 원가는 **꼬리(heavy) 사용자에서 초과**될 수 있다. pro 출력 단가는 $0.87/MTok로 flash의 3.1배(DeepSeek Docs)라 헤비 사용 시 객단가(3,500원 → 수수료 후 약 3,300원대)를 잠식할 수 있다.

**권고 R7**
1. **pro에도 호출별 `max_tokens` 상한과 합리적 상한선(detail 일일·summarize 빈도 등)을 유지**한다. "무제한"은 UX 표현일 뿐 백엔드에는 남용 방지 캡을 둔다.
2. **사용자별 월 원가 분포를 모니터링**해 상위 소수의 원가가 평균을 끌어올리는지 추적. 필요 시 공정사용(fair-use) 정책을 약관에 명시.
3. **pro 원가 가정도 실측으로 교체**한다(§1의 세션당 토큰·세션 수 가정은 추정).

### 6.9 종합 권고 — 우선순위와 의사결정

근거의 무게에 따라 우선순위를 매긴다.

**먼저 할 것(흑자 자체를 결정 — H 신뢰도 근거)**
1. **소진율·전환율을 베타의 1·2순위 KPI로 즉시 계측**(R1·R2). 두 값이 §2~§4 표 전체의 입력이므로, 측정 전 모든 손익 판단은 보수(100% 소진) 가정으로 고정.
2. **무료 한도는 "조이되 가치 절정 보장"으로 설계**(R1↔R2 균형). 가격 인상은 BEP를 거의 못 낮추므로 **1차 레버에서 제외**, 소진율·전환율 개선을 우선.
3. **비용 보호 인프라 상시화**: 모든 호출 `max_tokens` 상한, 캐시 적중률 KPI, 프로바이더 어댑터(폴백)(R3·R4·R7).

**런칭 전 반드시(비용 작음, 누락 치명 — H 신뢰도 근거)**
4. **제도 체크리스트 일괄 이행**(R6): AI 고지·라벨링, 통신판매업 신고+푸터, 개인정보 처리방침+위탁 명시, 출처 병기. 계도기간(~2027-01)은 유예일 뿐 면제가 아님.
5. **결제는 카드/간편 기본 + 연간권 설계**로 실효 수수료율 방어(R5). 국내 PG 확정(Stripe 불가), VAT·관리비를 마진표에 포함.

**상시 모니터링(외부·미확정 — 신뢰도 가변)**
6. **DeepSeek 단가·모델 ID 분기 재검증**(R3) — 메모리 단정 금지, 웹 출처로만. V4 deprecate 일정·폴백 발동 기준 사전 정의.
7. **freemium 전환율 벤치마크는 본 보고에서 [추정](신뢰도 낮음)**이므로, 실데이터 확보 시 §3·§4를 재산출. 목표 전환율은 BEP+안전마진으로 설정.

**최종 판단:** §4가 보여주듯 이 사업은 **BEP만 넘기면 마진이 가파르게 좋아지는 구조**(무료 풀 원가가 깔리고 유료 전환이 순익을 그대로 더함)다. 따라서 리스크 관리의 핵심은 "가격을 얼마로 할까"가 아니라 **① 무료 풀 원가를 소진율로 눌러 BEP를 낮추고 ② 전환율을 BEP 위로 끌어올리는 것** 둘이다. 나머지(단가·캐시·수수료·제도)는 이 두 레버의 완충재이거나 누락 방지 과제이지, 단독으로 흑자/적자를 가르지 않는다.

> 신뢰도 표기: 단가·요율·법요건은 §1~§5 인용 출처(asof 2026-06-27) 기반 H. 세션당 토큰·월 세션 수·무료/유료 비중·소진율·전환율 벤치마크는 모두 가정/[추정]으로 신뢰도 낮음 — 베타 실측으로 교체 전제.
