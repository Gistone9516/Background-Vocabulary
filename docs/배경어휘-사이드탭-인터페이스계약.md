# 배경어휘 사이드탭 — 프론트↔백엔드 인터페이스 계약 (SoT v1.0, 2026-06-25)

> buildflow ②단계 SoT. **충돌 시 이 문서가 권위(SoT wins).** 폴더별 spec은 여기에 conform.
> 상위 정본: 제품 동작·UX·필드 = `panel.html`(최신 기획). 인프라 제약(로케일·RAG·안전 게이트) = 백엔드.
> 코드 현재값 출처: `sidetab/packages/shared/{types,interfaces,pipeline.stub}.ts`, `prompts/index.ts`. 본 문서는 그 위의 **델타**.

---

## 0. 권위 규칙
- 화면 흐름·노출 필드·문구·태깅 UX = **panel.html이 권위**. 백엔드가 굽힌다.
- 로케일 라우팅·RAG 필수·고위험 거부·TTL·모델 라우팅 = **백엔드가 권위**. 프론트가 따른다.
- 프론트는 전부 동기 mock(SEED/MORE, setTimeout). 실제는 5엔드포인트 + SSE 스트리밍으로 분리.

## 0-1. ★ 이식성 불변 경계 (AWS Lambda 이식 대비 — 1순위 제약, 구현계획 §3-1)
> 이 프로젝트는 Cloudflare Workers 지금 / AWS Lambda 나중을 **어댑터 추가만으로** 가능해야 한다. 본 SoT의 어떤 변경도 이 경계를 깨면 안 된다. 빌드 에이전트는 이 절을 먼저 읽고 따른다.
- **core/·shared/ 안 런타임 전역 0건.** Workers 전용 바인딩(KV·Durable Objects·`caches.default`·`ctx.waitUntil`)·Node 전용 API 직접호출 금지. 외부 자원은 전부 `LlmClient·SearchProvider·CacheStore·EnvConfig` 인터페이스 뒤로 추상화하고 주입(DI).
- **웹표준만 사용:** fetch · Request/Response/Headers · Web Streams(ReadableStream·TransformStream) · Web Crypto · TextEncoder/Decoder · URL · AbortController/AbortSignal. (전부 Node 18+/Lambda 동작.)
- **런타임·공급자 특수성은 `adapters/`에만:** Workers env 매핑·Hono 부트·StreamEvent↔SSE 직렬화·tier 게이팅(헤더 검사)·Upstash/Tavily REST 호출.
- **Lambda 구현체는 지금 안 짠다.** 인터페이스 경계만 유지. `adapters/lambda/`는 트리거 시 추가(Hono 흡수, 50~100줄). 인터페이스를 Lambda 시그니처에 미리 넓히지 않는다.
- **이번 변경 점검 결과 = 런타임 누수 0:** enough/confidence·domain_risk·exclude·Source·Prompt5 3단 = 순수 데이터. signal = AbortSignal(웹표준). /summarize 게이팅 = 어댑터 라우트. site = URL 호스트 파생(웹표준).
- **하드플로어 가드(사이클1 신규):** `core/`·`shared/`가 런타임 전용 모듈 import 또는 런타임 전역 참조 시 빌드 실패하는 경계 테스트 추가.

## 2. 공유 타입 변경 (shared/ 델타)

### 2-1. 신규 타입 `Source`
```ts
export interface Source { title: string; site: string; url: string }
// site = URL 호스트에서 파생(어댑터/core 책임). RagDoc에는 site 없음 → 변환.
```

### 2-2. `Term` (P3 출력) — 필드 추가
```ts
// 추가: group?  (그룹뷰 toggle용, 프론트 t.group)
group?: string;
// 기존 relates_to 유지(프론트 'relates'는 relates_to로 통일 — 프론트가 백엔드명 채택)
```
- `kind`는 자유 문자열 유지(프론트 KIND_HUE 7종은 표시 힌트일 뿐 제약 아님).
- `tag`는 항상 "몰라"로 생성(기존 유지). 미태깅 상태는 프론트 전용(§4-1).

### 2-3. `Prompt2Out` — 종료 신호 추가 (D1)
```ts
export interface Prompt2Out {
  question: string;
  choices: Choice[];
  enough: boolean;     // 신규: true면 더 묻지 말고 recommend로(백엔드 판단)
  confidence: number;  // 신규: 0..1, UI 힌트("거의 다 좁혔어요")
}
```
- **엔벨로프 규칙(프론트 가드):** 최소 3질문 전에는 `enough=true`여도 계속, 8질문 도달 시 강제 종료. 즉 백엔드가 [3,8] 안에서 결정.
- 진행 UI: 고정 분모 제거. "AI가 좁히는 중 · N번째" + confidence 높으면 힌트. 점은 누적형.

### 2-4. `Prompt5In` / `Prompt5Out` — 구조 재정의 (D2)
```ts
export interface Prompt5In {
  term: string; kind: string; area: string; job_type: JobType[];
  domain: string; topic: string; locale: Locale;  // 신규: recommend RAG 캐시 재사용 키
  deepen?: boolean;                                 // 신규: "더 깊이(예시·비유 추가)"
}
export interface Prompt5Out {           // 프론트 상세 3단 구조로 재정의
  what: string;        // 개념 (프론트 t.what)
  whymine: string;     // 내 맥락 (t.whymine)
  how: string;         // 활용 (t.how)
  misc?: string;       // 보조 콜아웃 (t.misc)
  related: string[];   // 함께 볼 어휘 (일반 관련어, relates_to와 별개)
  sources: Source[];   // 출처 필수. 빈 배열이면 프론트 "확인된 출처 없음" 폴백
  example?: string;    // deepen 시 추가 예시
}
// 폐기: definition, detail (what/whymine/how로 대체)
```

### 2-5. `RecommendInput` — 페이지네이션 + 안전 + 취소
```ts
export interface RecommendInput extends Prompt3In {
  domain: string; topic: string; locale: Locale;
  domain_risk: DomainRisk;   // 신규: LLM 판정 고위험을 recommend서 재차단(판단대기 #2)
  exclude?: string[];        // 신규: 이미 보여준 term명. 다음 우선순위 배치(D4)
}
```

### 2-6. `Pipeline` 시그니처 — AbortSignal (판단대기 #1)
```ts
recommendStream(input: RecommendInput, signal?: AbortSignal): ReadableStream<StreamEvent>;
// core/llm: streamTerms(req, signal?) 로 전파. workers: 요청 signal 연결.
```

## 5. 화면별 계약 상세
- **entry:** `S.input`→raw_input. `S.cond`(조건 추가)→user_condition. context_object는 명시적 붙여넣기 시만(현재 별도 필드 없음 → MVP 빈값). 고위험 정규식 1차 통과해도 /classify domain_risk가 권위.
- **narrow:** /classify 첫 질문 → /next 루프. 정적 stepDefs 트리는 mock이라 폐기, 질문은 전부 백엔드 생성. 다중선택 2개·되돌리기 → Prompt2In.history `{label, action:'선택'}`. "지금 충분해요" = 수동 조기 종료(enough 무시하고 recommend).
- **terms:** /recommend SSE, term 도착마다 stagger(`_new`). "더 보기" → exclude=현재 term명들로 2차 /recommend. groupView=t.group. 태깅은 클라 상태 + chrome.storage.local('known' 영속).
- **summary:** 기본 = 클라 buildSummary(즉시). "AI로 더 정리"(유료) = /summarize에 vocab+tag+condition 실어 호출.
- **paywall:** free 주7회(FREE_WEEKLY_LIMIT). 429 시 limitHit UI. remaining 표시는 클라 추정(게이팅 Tier3 전까지 cosmetic).
- **refusal:** classify domain_risk=high 또는 프론트 정규식. **부드러운 거부** — 차단만 하지 않고 왜 못 다루는지 이유를 설명하고 재시도(다른 맥락으로) 유도. 사용자向 문구 정본 = panel.html `screenRefusal`("이 분야는 도와드리기 어려워요 / 의료·법률처럼 개인의 진단·판단이 필요한 고위험 영역은 다루지 않아요. 기술·창작·비즈니스 학습 맥락에서 다시 시도해 주세요"). 백엔드 HIGH_RISK_REFUSED는 신호일 뿐, 표시 문구는 프론트 화면이 권위.

## 6. 폴더별 변경 체크리스트 (③ 빌드 분할 입력)
- **shared/**: types.ts(Source, Term.group, Prompt2Out, Prompt5In/Out, RecommendInput), pipeline.stub.ts(recommendStream signal), interfaces.ts(LlmClient.streamTerms signal?). prompts/index.ts(P2 enough·confidence 지시, P3 exclude·group 지시, P5 3단+sources 지시).
- **core/llm**: streamTerms(req, signal?) 취소 전파.
- **core/pipeline + rag + locale**: nextBranch enough/confidence, recommendStream(signal)·exclude·domain_risk 재게이트, detail RAG 캐시 재사용(domain+topic+locale→sources 추출, site=호스트 파생).
- **adapters/workers**: /next·/recommend·/detail·/summarize 라우트 갱신. /summarize x-tier=paid 게이팅. 요청 signal→recommendStream. StreamEvent→SSE 유지.
- **extension/**: panel.html 5화면 React 이식 + 위 계약으로 배선(가장 큰 작업, 별 사이클).

## 7. 열린 항목 (구현 중 점검)
- **D2 출처 근거 정확도(정밀 우선, 레드펜 3=ⓐ):** detail이 recommend RAG 캐시(rag:domain:topic:locale) 재사용해 term별 관련 출처 추출. **확신 가는 귀속만 포함, 애매하면 `sources:[]`(빈 배열)로 두고 프론트 "확인된 출처 없음" 폴백** — 거짓 귀속 금지(틀린 출처가 신뢰를 더 깎음). 매핑 정확도 실측, 부족하면 term별 경량 재검색. `site`는 URL 호스트 파생(RagDoc엔 없음).
- **exclude 페이지네이션:** P3 프롬프트가 "이미 제시한 어휘 제외, 다음 우선순위"를 일관 준수하는지 검증(중복·고갈 처리).
- **enough 캘리브레이션:** [3,8] 엔벨로프에서 confidence 임계 실측(너무 일찍/늦게 종료 방지).
- **remaining 정확도:** 익명 게이팅(설치 UUID, Tier3) 전까지 클라 추정.
