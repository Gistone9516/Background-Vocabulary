## 1. 한 줄 정체

AI에 말 걸기 전, 그 분야 핵심 어휘(말그릇)를 클릭으로 쥐여주는 **크롬 확장(MV3)**. 타깃은 자기 전공 밖으로 무언가 만들려는 교차전공 빌더(탑다운). 백엔드는 LLM·검색·캐시를 묶는 스트리밍 프록시.

## 3. 아키텍처 원칙 — 이식성이 1순위 제약

### 3-1. ★ 절대조건: 비즈니스 로직에 런타임 전용 바인딩 0건
Workers 전용 바인딩(KV·Durable Objects·`caches.default`·`ctx.waitUntil`)을 라우트/도메인 로직에서 **직접 호출하지 않는다.** 모든 외부 자원(캐시·검색·LLM·설정)은 인터페이스 뒤로 추상화하고 주입(DI)한다. 이걸 첫날부터 강제하지 않으면 코드가 커지며 바인딩이 새어들어 "얇은 어댑터" 약속이 통째 재작성으로 붕괴한다(검증 결과 최대 리스크).

**강제 수단:** 모듈 경계 lint 규칙(또는 의존성 테스트)으로 `core/` 안에서 런타임 전역 접근 시 빌드 실패.

> **이식성 = 인터페이스 경계는 지금, 구현체는 나중.** 인터페이스는 지금 긋되 **AWS Lambda 구현체(`adapters/lambda/`)는 지금 짜지 않는다**(트리거 없음·Hono 흡수로 50~100줄, 필요 시점에). 인터페이스를 Lambda 시그니처에 미리 맞춰 넓히지 않는다.

### 3-2. 런타임 어댑터 경계 (얇게, 50~100줄)
| 부분 | Workers (지금 구현) | AWS Lambda (트리거 발생 시) |
|---|---|---|
| 진입 | `fetch(request, env, ctx)` | `streamifyResponse((event, responseStream, ctx))` + event→Request 변환 |
| 설정 주입 | `env` 바인딩 → 인터페이스 구현 | `process.env` + SDK → 같은 인터페이스 구현 |
| 스트리밍 출력 | `new Response(readable)` | `readable.pipeTo(responseStream)` |
| 후처리 | `ctx.waitUntil` | 핸들러 반환 전 await, 또는 SQS/EventBridge |

Hono가 진입·스트리밍 어댑터 대부분을 흡수한다.

### 3-3. 웹표준만 사용 (이식 가능 확인됨)
Fetch, Request/Response/Headers, Web Streams(ReadableStream·TransformStream), Web Crypto, TextEncoder/Decoder, URL — 전부 Node 18+ 동작.

## 5. 스트리밍 아키텍처 (end-to-end)

1. **사이드패널 페이지**가 Workers로 `fetch(url, {headers:{Authorization}, signal})`. 응답 SSE 바이트를 `getReader()`로 읽어 **StreamEvent로 파싱**해 term 단위 DOM 갱신. 패널이 닫히면 `AbortController.abort()`로 fetch 취소.
2. **Workers**가 DeepSeek로 `stream:true` 요청 → 업스트림 `response.body`를 변환 통과시켜 클라이언트로 파이핑. **파이프는 await하지 않는다**(fire-and-forget). `: keep-alive` 처리.
   - **★ 취소 체인(사이클2):** 클라이언트 fetch 취소 → Workers readable의 `cancel` → **업스트림 DeepSeek fetch를 `AbortController`로 중단**. 이 체인을 빠뜨리면 패널 닫힘마다 업스트림 고아 연결이 남는다.
3. **★ 타입 경계 변환 책임(사이클2 — 모순 제거):** 단계별로 책임을 못 박는다.
   - `core/llm`: DeepSeek SSE **`Uint8Array` → `StreamEvent`** 파싱·변환을 내부에서 완결(`LlmClient.stream()`이 `ReadableStream<StreamEvent>`를 내보냄).
   - `adapters/workers`: **`StreamEvent` → SSE 바이트**(`event:…\ndata:…\n\n`) 직렬화해 `Response`로.
   - `extension/sidepanel`: **SSE 바이트 → `StreamEvent`** 파싱.
   세 변환을 분리 명시해야 A(core/llm)와 C(workers)가 같은 타입을 다르게 해석하는 통합 충돌을 막는다.
4. **★ term 단위 경계:** Term JSON 객체 하나가 완성되면 즉시 `{type:'term'}`으로 flush(누적 배열 전송 금지). **JSON mode 스트림에서 이 경계 파싱이 실제로 동작하는지는 G1에서 선검증**(§10).
5. **★ 에러 채널:** 변환·파이핑 중 업스트림 오류(429·5xx·연결 끊김·예외)를 잡아 **`{type:'error', code, message}` enqueue 후 `controller.terminate()`**. 200 헤더가 이미 나간 뒤라 SSE 내부 error 이벤트가 유일한 전달 수단. 사이드패널은 error 이벤트로 에러 UI + 재시도 표시.
6. **SSE 청크 경계 != 이벤트 경계** → 양쪽에서 부분 라인 버퍼링 후 파싱.

## 6. 로케일 라우팅 + 도메인 안전 (확정 설계)

**원칙: 검색 언어는 검색 전에 단 한 번 결정론적으로 정한다. 투기적 2회 검색·품질 재평가 금지.**

```
[진입 분류 콜(프롬프트 1/2)] — 출력에 domain(열거 키)·search_locale·domain_risk 동시 산출
   ├─ 0) ★고위험 게이트: domain_risk='high'(의료·법률 개인판단 등) → 진행 안 함, 안전 거부/범위 안내 (기획 §11)
   ├─ 1) 정적 도메인→로케일 맵 조회. 이 맵의 키 = §7 캐시키 도메인 열거 + hard_domain 플래그를 겸함(삼중 역할).
   │      en 예: pid_control/robotics/software/startup │ ko 예: local_smb_policy/real_estate_lease
   │      hard_domain:true 예(초기): financial_modeling/payment_settlement/ar_vr → flash여도 pro 폴백(G6 실측 보정)
   └─ 2) 맵 미스 → 분류 규칙(우선순위, 첫 매치): ① ko 한국 관할·제도 ② ko 한국 로컬 시장·관행 ③ ko 한국어 토착 용어 ④ en(기본·지배 경로)
[검색은 확정된 단일 로케일로 1회만 실행 → flash/pro는 hard_domain·tier로 결정]
```

- **★ 모델 라우팅(사이클2 명시):** 무료=flash, 유료=pro 기본. 단 `hard_domain:true` 도메인은 flash 사용자라도 pro 폴백(빌더 검증서 flash+RAG 경계선 도메인). 판정은 정적 맵 플래그 참조(별도 모듈 아닌 pipeline.ts 인라인).
- **결정성의 한계:** "결정론"은 정적 맵 경로에만 엄밀. 맵 미스 LLM 분류는 확률적 → 정적 맵 커버리지 ↑가 1차 수단, 오분류 임계는 G4 수치(§10).
- **★ ko 오분류·미구현 안전장치(사이클2):**
  - LLM이 ko로 분류해도 **Tavily 한국어 쿼리는 SearchProvider(Tavily 구현체)에서 throw 가드로 차단**.
  - **네이버 구현 전(MVP):** ko 분류 케이스는 영어 RAG로 폴백하되 "근거 제한" 표기(한국 고유 주제는 RAG 약화 감수), 또는 저위험이면 noRAG 진행. 네이버 정식 채택 시 ko 1차 공급자로 전환(§12).
- **★ 예외 경로(재시도 트리거 아님, §7과 일치):** 검색 0건·API 오류·타임아웃 → ⓐ캐시 히트가 있으면 (stale라도) 캐시 근거로 진행("근거 제한" 표기) → ⓑ캐시 없으면 저위험은 근거 제한 경고 후 진행 또는 에러 스트림, 고위험은 거부. **noRAG 신규 생성을 기본 경로로 삼지 않는다**(§7 RAG 필수와 일치). 언어를 바꿔 재검색하지 않는다.
- **bilingual(혼합 주제 2개 검색):** MVP 보류(관찰 후 추가).

## 7. RAG 파이프라인

```
classify(domain[열거], locale, domain_risk, hard_domain)
  → ★모든 tier(flash·pro) RAG 필수 — noRAG 정확도 68~75로 기준 미달, 무료 tier도 RAG 생략 금지 (기획 §11)
     단, 검색 API 장애 시에 한해 §6 예외 경로(캐시 폴백 우선)를 따른다 — 정상 운영 중 RAG 생략은 금지.
  → cacheKey = hash(domain[열거 키], normalizeTopic(topic), locale)
  → CacheStore.get → hit: grounding_facts 재사용 / miss: SearchProvider.search → 사실 추출 → set(…, TTL)  ※오류·0건 미캐시
  → 프롬프트3에 grounding_facts 주입 → DeepSeek 생성(스트리밍)
```

- **★ 캐시키 정규화 코드화:** `domain`은 프롬프트1이 정적 맵 **열거 키**로 출력(자유문장 금지, G1에서 OOV 거동 선검증). `topic`은 `normalizeTopic()`(소문자화·공백/조사 strip) 순수함수를 거쳐 해시. 정규화 책임을 LLM이 아닌 코드에. **위치 = `shared/utils.ts`**(types.ts와 분리해 읽기전용 원칙 보존).
- **★ 동의어 한계(사이클2):** `normalizeTopic()`은 형태소·동의어('스타트업 투자' vs '펀딩')까지 수렴시키지 못함. 작은 정적 alias 맵으로 보강 가능하나, **지금 박지 않고 G2 캐시 적중률 실측 후 분산이 크면 5~10쌍 추가**(추측 선구현 회피). 훅만 utils에 남김.
- **★ TTL 규칙:** 도메인 유형별 기본값(기술/창작/비즈니스 24시간, 시사/정책 1시간). `set`에 TTL 미지정·0이면 throw. RAG 기여 본질이 최신성보다 커버리지라 길게.
- **공통 고정구간 캐시:** 프롬프트1·2의 동일 도메인 첫 1~2 분기 선택지 캐시(기획 §3-2)는 RAG 도메인 캐시와 별개 키 공간.
- RAG 기여 = 누락 핵심어 커버리지(검증 결론). 프롬프트3 SYS 커버리지 규칙(P31) 유지.

## 8. 핵심 인터페이스 (SoT — opus가 병렬 착수 전 먼저 확정)

```ts
type ModelId = string;                 // 리터럴 union 금지 — fallback 모델 교체 시 SoT 수정 불필요. 허용목록 검증은 adapters에서만.
type DomainRisk = 'low' | 'high';
type StreamEvent =                     // C·D 양쪽이 이 타입에 의존 — 자의 정의 금지
  | { type: 'term'; term: Term }
  | { type: 'done' }
  | { type: 'error'; code: string; message: string };

interface SearchProvider {             // ko 한국어 쿼리는 구현체(Tavily)에서 throw 가드
  search(q: { query: string; locale: 'en'|'ko'; depth: 'basic'|'advanced'; maxResults: number; rawContent: boolean }): Promise<RagDoc[]>;
}
interface CacheStore {                 // Upstash REST. 항상 TTL(미지정 시 throw)
  get(key: string): Promise<string | null>;
  set(key: string, val: string, ttlSec: number): Promise<void>;
}
interface LlmClient {                  // DeepSeek 구현, thinking OFF 기본.
  // ★ stream()은 ReadableStream<StreamEvent>를 내보낸다. DeepSeek SSE의 Uint8Array→StreamEvent 변환은
  //    core/llm 내부에서 완결한다(어댑터는 StreamEvent→SSE 바이트 직렬화만). §5-3 경계 책임 참조.
  stream(req: { model: ModelId; messages: Msg[]; jsonMode?: boolean }): ReadableStream<StreamEvent>;
}
interface EnvConfig {                  // 런타임별 주입
  deepseekKey: string; tavilyKey: string; upstashUrl: string; upstashToken: string;
}
```

- **사용량 카운트:** 인터페이스로 추상화하지 않는다(과적합). `core/pipeline.ts`에서 Upstash REST를 **인라인 호출**(예: `sub:{userId}:week`). 단 **userId 파생 전략 미결(§12)로 게이팅 구현 자체는 Tier3 보류**.
- **★ 프롬프트 I/O 계약 출처:** 프롬프트 5종의 입출력 타입(Choice·Term 스키마, `condition_required`·`gap_type`·`relates_to`·`job_type` 9종 열거, P27~P31 조건부 트리거 등)의 **단일 정본은 기획 §5·§5-1**. `shared/prompts/`는 그걸 옮겨 담을 뿐, 0단계 SoT 작성 시 §5-1을 그대로 따른다(재해석 금지).
- **결제 PG 인터페이스:** PG 확정 후 추가(현재 보류).

## 9. 빌드 태스크 분해 + 의존 순서 (buildflow 입력)

**0단계 — SoT 선작성 (opus 단독, 직렬):** `shared/types.ts`·`utils.ts`(normalizeTopic)·`interfaces.ts`·`prompts/`(기획 §5-1 전사)·**`pipeline.stub.ts`**(pipeline의 exported 함수 시그니처). **SSE fixture 2종(happy-path + error-path `{type:'error',…}`)**. 
> **0단계 완료 조건(병렬 허가 게이트):** ① G1·G4·G7 통과(§10) ② 기획 §5-1의 모든 조건부 필드 트리거(direction/use_example/context_note/relates_to/order 활성화 조건)가 `shared/prompts/` 타입 주석에 1:1 반영됐는지 opus 체크리스트 확인.

**1단계 — 병렬 구현 (sonnet, 폴더별 1 에이전트, 워크트리 격리, `shared/` 읽기 전용, git 명령 금지):**

- B. `core/rag` + `core/locale` + **`core/pipeline.ts` 골격 소유** — 파이프라인·캐시(정규화·TTL)·로케일·고위험 게이트·flash/pro 인라인 라우팅·사용량 카운트 인라인. SearchProvider/LlmClient/CacheStore 인터페이스만 호출.
- C. `adapters/workers` — Hono 부트 + env 매핑 + **StreamEvent→SSE 직렬화** + `usageCounter.ts`(Upstash). `pipeline.stub.ts` 시그니처에만 의존.
- D. `extension/sidepanel` — **chrome.sidePanel React 페이지** + SSE→StreamEvent 파싱·렌더 + 에러 UI. SSE fixture(happy+error) 목으로 독립 개발. Vite는 MV3 manifest 연동. **★기획 §4/§8의 웹앱 SPA 표기는 구버전 — §11(크롬 확장)이 정본, 참조 금지.**
- F. `adapters/search-providers` — **Tavily(영어) 구현 + SearchProvider 인터페이스만**. ko 한국어 쿼리 수신 시 throw 가드. **네이버(한국어) 구현은 §12 보류**(MVP ko 히트 미검증).

> **프롬프트 5종 소유권:** 템플릿 조립·호출·파싱은 **`core/pipeline.ts`(B 소유) 단독**. 서브모듈은 인터페이스만 호출(중복 구현·계약 재해석 충돌 방지).

**2단계 — 통합 (opus):** 인터페이스 결선·e2e 스트리밍(정상+에러+취소)·CORS·배포(wrangler).
