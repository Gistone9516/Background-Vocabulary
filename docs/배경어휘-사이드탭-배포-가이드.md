# 배경어휘 사이드탭 — 워커 배포 가이드 (Cloudflare)

> 로컬 라이브 통합(실 API)은 이미 검증 완료. 이 문서는 **프로덕션 배포**(Cloudflare Workers)만 다룬다.
> 배포는 Cloudflare 계정 인증이 필요해 사람이 직접 한 번 로그인해야 한다(헤드리스 불가). 로그인만 끝나면 나머지는 그대로 따라 하면 된다.

## 0. 사전 확인
- Cloudflare 계정 필요. Workers는 **유료 플랜**이 필요하다(`wrangler.toml` 주석대로 LLM 프록시는 CPU 10ms 무료 플랜으로 부족).
- 작업 위치: `sidetab/packages/adapters/workers`
- 키 4종은 로컬 `.dev.vars`에 있다(커밋 금지). 프로덕션은 아래 `wrangler secret put`로 따로 등록한다.

## 1. 로그인 (사람이 직접 1회)
```bash
cd sidetab/packages/adapters/workers
npx wrangler login        # 브라우저 OAuth 창이 뜸. 승인하면 끝.
npx wrangler whoami       # 인증 확인
```

## 2. 시크릿 등록 (키 4종)
`.dev.vars`의 값과 동일하게 넣는다. 각 명령이 값 입력을 물어본다.
```bash
npx wrangler secret put DEEPSEEK_API_KEY
npx wrangler secret put TAVILY_API_KEY
npx wrangler secret put UPSTASH_REDIS_REST_URL
npx wrangler secret put UPSTASH_REDIS_REST_TOKEN
```

## 3. 배포
```bash
npm run deploy            # = wrangler deploy
```
배포가 끝나면 출력에 실제 URL이 찍힌다. 보통 `https://sidetab-api.<계정서브도메인>.workers.dev` 형태다. 이 URL을 아래 4단계에 쓴다.

## 4. 프론트 배선 (배포 URL 확정 후)
배포 도메인을 두 곳에 반영해야 확장이 프로덕션 워커와 통신한다.

1. **확장 빌드의 WORKER_BASE**: 빌드 시 환경변수로 주입한다.
   ```bash
   cd sidetab/packages/extension
   VITE_WORKER_BASE=https://sidetab-api.<계정서브도메인>.workers.dev npm run build
   ```
   (env 없이 빌드하면 `api.ts`의 placeholder `https://sidetab-api.example.workers.dev`로 나가 동작 안 함.)

2. **manifest host_permissions**: `sidetab/packages/extension/manifest.json`의 placeholder를 실제 도메인으로 교체한다.
   ```
   "https://sidetab-api.example.workers.dev/*"  →  "https://sidetab-api.<계정서브도메인>.workers.dev/*"
   ```
   로컬 개발용 `http://127.0.0.1:8787/*`, `http://localhost:8787/*`는 그대로 둬도 무방하다(프로덕션 무해).

## 5. CORS 확인
워커 CORS(`src/index.ts`)는 `chrome-extension://`와 `http://localhost`·`http://127.0.0.1`을 허용한다. 확장에서의 fetch는 host_permissions 허용 도메인이면 CORS 없이도 통과한다. MVP는 광역 허용이며, 도메인 확정 후 origin을 좁힐 수 있다.

## 6. 실 확장 로드 확인
```
chrome://extensions → 개발자 모드 → 압축해제 로드 → sidetab/packages/extension/dist
```
사이드패널을 열어 entry→narrow→terms(Keep)→담은 어휘→이전 탐색 복원 흐름을 프로덕션 워커로 확인한다.

## 참고 — 로컬 라이브 통합(이미 검증됨)
배포 없이 로컬에서 실 API로 돌리는 방법(검증에 사용).
```bash
# 워커(실 키, .dev.vars 자동 주입)
cd sidetab/packages/adapters/workers && npm run dev      # 127.0.0.1:8787

# 확장 dev 서버(실 워커 연결)
cd sidetab/packages/extension && VITE_WORKER_BASE=http://127.0.0.1:8787 npx vite --port 5174
# 브라우저로 http://localhost:5174/sidepanel.html
```
