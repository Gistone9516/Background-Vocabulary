## 2. ★ 메인 작업 — Keep 전환 (방향 확정, 사용자 directive 2026-06-25)

### 2-1. 배경·결정(왜 바꾸나)
- **현 어휘 카드 태깅 {알아·몰라·적용 모름} = 포지션 애매.** 매 어휘마다 자기평가를 강요하는 마찰 대비 가치가 약하다(메타인지 의도였으나 잡일에 가까움). → **축소·대체**.
- **정리 보기(paste-prompt) = 제품 종착점과 어긋남.** 종착점은 "복사용 프롬프트 생성"이 아니라 "사용자가 분야 개념을 이해해 스스로 말할 수 있게 되는 것"(메모리 `sidetab-project-status` 종착점 교정 참조). paste는 부수 편의일 뿐. → **재배치·격하**.
- **대안 = Keep(담기).** 카드를 북마크하듯 담는다. 마찰 낮고 가치 명확(유용한 어휘를 큐레이션). 진입 시 입력한 텍스트를 **인덱스**로 삼아, 그 인덱스를 클릭하면 그때 Keep 한 카드 리스트를 다시 본다. = **"내 배경어휘 모음"을 축적** → 제품 목적("배경지식 어휘력을 늘려 AI를 잘 쓰기")과 정합.
- **opus 의견(권고):** 위 방향에 동의. 태깅은 제거하고 Keep 단일 동작으로 대체, 정리보기는 "담은 어휘" 뷰로 재배치하는 것을 권한다. 단 "완전 제거 vs 흔적 남김"은 2-5 열린 질문에서 사용자 확정.

### 2-2. 데이터 모델 (클라이언트 저장)
- 저장소: `chrome.storage.local`(확장), 개발(vite dev)은 `localStorage` 폴백 — 기존 api.ts `getUserId`의 분기 패턴 그대로.
- 스키마(예):
  ```ts
  interface KeptTerm { term; one_line; why; kind; priority; group?; detail?: Prompt5Out } // 카드 핵심 + 상세 캐시(있으면)
  interface KeptSession { id: string; input: string; domain: string; createdAt: number; terms: KeptTerm[] }
  // 저장 키: "kept_sessions" -> KeptSession[]  (또는 input 해시 키로 맵)
  ```
- 기존 localStorage `'known'`(아는 어휘) 저장 로직은 Keep 저장으로 대체·정리.

### 2-3. 화면 변경
- **terms(화면2):** 카드의 3-way 태그 버튼 → **Keep 토글 1개**로 대체. "알아=숨김 / 몰라=강조 / 적용모름" 분기 로직 제거. Keep 누르면 현재 인덱스(현 세션 input)에 그 카드 저장, 다시 누르면 해제. footer "담은 어휘 N"은 Keep 개수로 자연스럽게 의미가 맞음.
- **entry(진입):** 컴포저 아래에 **과거 인덱스(이전 입력) 리스트** 노출. 항목 클릭 → 그 세션에서 Keep 한 카드 뷰로 이동. (위치 대안: 헤더에 히스토리 아이콘 — 2-5 확인.)
- **kept 뷰(신규 또는 summary 재활용):** 한 인덱스의 Keep 카드 목록을 보여주는 화면. 카드 펼쳐 상세(이미 있는 detail/출처 재사용). 여기서 Keep 해제도 가능.
- **summary(화면3) 재배치:** "정리 보기" → "담은 어휘 보기"(현재 세션 Keep 목록)로 전환하거나 제거. paste/공유와 "AI로 더 정리"(유료 /summarize)는 옵션으로 격하 또는 제거 — 2-5 확인.

### 2-4. 백엔드 영향
- Keep은 전부 클라이언트 저장 → **백엔드(워커·프롬프트) 무변경**. 이식성 가드 영향 없음.
- 태깅 제거 시 정리 대상(미사용화): `App.tsx`의 UITag·UI_TO_TAG·setTag·태그 관련 UI, `Prompt4In.vocab.tag` 사용처. **계약(types.ts) 자체는 건드리지 말 것**(SoT, 다른 곳 의존). 프론트 사용만 제거.
- paste/AI정리 제거 시 `/summarize` 엔드포인트는 미사용으로 남김(삭제하지 말고 보류 — 라우트·게이팅 유지). 인터페이스계약 §1·§5의 summary 행이 "보류"임을 주석.

### 2-5. 열린 질문 (실행 세션이 사용자에게 확인 후 진행)
1. 태깅: **완전 제거** vs Keep + 최소 흔적(예: 아는 것 접기만 유지)?  → 권고: 완전 제거.
2. 정리보기/paste/AI정리: **완전 제거** vs 격하 유지(접힌 옵션)?  → 권고: "담은 어휘" 뷰로 대체하고 paste는 보조 버튼으로 남김.
3. 인덱스(과거 입력) 리스트 위치: entry 하단 vs 헤더 히스토리 아이콘?
4. Keep 영속 범위: 브라우저 storage 한정(계정 동기화는 후속 Tier).
5. 같은 입력으로 또 들어오면: 기존 인덱스에 누적 vs 새 인덱스 분리?
