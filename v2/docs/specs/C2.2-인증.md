# C2.2 스펙 — 인증 (buildflow ②, SoT §3-2·§8·§11에서 파생)

> 상위 정본 = `../인터페이스계약-v2.md`. v1 §8 이월 원본 = `v1/…/adapters/workers/src/auth/{jwt,google}.ts`·`db/d1-user-repo.ts`·`src/index.ts`(/auth 라우트). 무손실 이식(로직 verbatim, 런타임만 교체).

## 0. 범위 / 비범위
- **범위**: 자체 JWT(HS256·Web Crypto)·Google OAuth PKCE 코드교환·엔타이틀먼트·UserRepository(PG)·`/auth/{google,refresh,logout}`·`/subscription/status`·resolveTier 미들웨어·JWT 파생 user_id로 CRUD 인증 교체(C2.1 x-user-id 대체).
- **비범위(이월)**: 실 Google 크레덴셜 셋업·실 OAuth end-to-end = 핸즈온(로컬은 MockGoogleOAuthClient로 검증). 결제/구독 활성(/checkout 실결제) = C5. 게이팅 한도(주간·IP) = C2.3.

## 1. 핵심 설계 결정
### 1-1. 계층 배치 (순수 로직 vs 공급자 vs 부트)
- **core/auth** (순수, Web Crypto 허용): `jwt.ts`(sign/verify/issue, v1 verbatim)·`entitlement.ts`(effectiveEntitlement)·`auth-service.ts`(loginWithGoogle/refresh/logout/status — 주입된 포트만 호출). 런타임 전역 0.
- **@vock/providers** (신규, 공급자 어댑터, 웹표준 fetch): `google-oauth.ts` = RealGoogleOAuthClient(Google 토큰 엔드포인트, platform별 client_id/secret 선택). local·aws 공유. (C2.4의 DeepSeek·Tavily·Upstash도 여기로.)
- **persistence**: `PgUserRepository`(d1-user-repo → SqlRunner 이식)·`PgJtiBlacklist`·migration `0002_auth.sql`(revoked_jtis). users 테이블은 0001에 이미 존재.
- **http-app**: `/auth/*` 라우트(authService 배선)·resolveTier/requireAuth 미들웨어·CRUD user_id를 JWT sub에서 파생.
- **local**: MockGoogleOAuthClient(고정 신원, 실 Google 없이 로그인 흐름 검증) + authConfig(DEV 시크릿) 배선.

### 1-2. 블랙리스트 저장소 재결정(§11)
- **Aurora 테이블(revoked_jtis) 채택** — v1 D1 결정의 자연 이관. 근거: ①durable·Data API 조회 가능 ②Upstash 장애와 무관하게 revocation은 fail-closed(보안 우선) ③refresh는 저빈도라 테이블 부하 무시 가능. TTL 대체 = `expires_at` 컬럼 + 만료행 무시(주기 청소는 C2.5 EventBridge 후보). refresh 검증 시점에만 조회(access는 networkless 유지).

### 1-3. 시각 단위 경계(승계)
- JWT=초(epoch sec), DB·도메인=밀리초. 변환은 jwt.ts(초) / 리포(ms) 각자. AccessTokenClaims/RefreshTokenClaims=초.

### 1-4. Google 서명 재검증 생략(승계)
- id_token을 Google 토큰 엔드포인트에서 TLS 직결 수신하므로 서명 재검증 생략, 대신 iss/aud/sub/email 검증(v1 google.ts 그대로). PKCE code_verifier 필수.

## 2. 타입·포트 (shared, C1에서 이월분 이식)
- types/auth.ts: `SubscriptionStatus`·`PgProvider`·`Currency`·`UserRecord`·`Entitlement`·`AccessTokenClaims`·`RefreshTokenClaims`·`NewUser`·`EntitlementPatch` (v1 types.ts §8 + interfaces.ts).
- ports/auth-ports.ts: `UserRepository`(findById/findByEmail/findByGoogleSub/create/applyEntitlement)·`GoogleOAuthClient`(exchange(code,verifier,redirectUri,platform)→GoogleIdentity)·`JtiBlacklist`(isRevoked/revoke)·`GoogleIdentity`·`AuthConfig`(jwtSecretCurrent·jwtSecretPrev?·jwtKid·devForceTier?).

## 3. 엔드포인트 (SoT §3-2, v1 오케스트레이션 이식)
| 엔드포인트 | 동작 | 에러코드(승계) |
|---|---|---|
| POST /auth/google | {code,code_verifier,redirect_uri,platform} → exchange → findByEmail∥create → issueTokens | AUTH_UNAVAILABLE 503·AUTH_FAILED 401 |
| POST /auth/refresh | {refresh_token} → verifyRefresh → **blacklist 검사** → DB 최신 tier 재조회 → issueTokens | TOKEN_REVOKED 401 |
| POST /auth/logout | {refresh_token} → verifyRefresh → **jti revoke** → 204 | — |
| GET /subscription/status | Bearer → verifyAccess → DB 재조회 → {tier,status,expires_at,access_token 재발급} | AUTH_REQUIRED 401·TOKEN_EXPIRED 401 |
- resolveTier(networkless): DEV_MODE+DEV_FORCE_TIER override → Bearer verifyAccess → claims.tier, 없으면 free. x-tier류 헤더 완전 무시(C-21).
- requireAuth(CRUD): Bearer verifyAccess 필수 → sub를 user_id로. 실패 401. (C2.1 x-user-id 경로 제거.)

## 4. 패키지·의존 방향
- 신규 `@vock/providers`(→shared). persistence +auth 리포. core +auth(→shared). http-app는 authService(core)+포트(shared). local +providers(Mock는 local, Real은 providers).
- boundary ALLOWED: providers:[shared] / core:[shared] / http-app:[shared,core] / local:[shared,core,http-app,persistence,providers].

## 5. 로컬 검증 (완료 기준)
- e2e-auth: MockGoogle 로그인 → access/refresh 수령 → **access로 CRUD**(x-user-id 없이 세션 생성·조회) → refresh로 재발급 → logout(revoke) → 같은 refresh 재사용 시 401 → 만료/위조 토큰 401 → resolveTier가 paid 클레임 반영.
- 기존 게이트(mock·pg) 무회귀. CRUD e2e는 JWT 헤더로 갱신(x-user-id 제거).

## 6. 열린 항목
- revoked_jtis 만료행 청소 주기 = C2.5(EventBridge) 후보. 실 Google client_id 이원화(web/desktop) 콘솔 실등록 = 핸즈온(F6 중하 신뢰).
