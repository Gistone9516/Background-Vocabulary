# C3 스펙 — 웹·랜딩 (buildflow ②, SoT §1·§7에서 파생)

> v1 UI 자산 = `v1/…/extension/sidepanel/`(App.tsx 1534행·theme.css 564행·i18n.ts 1055행·examples.ts). theme.css 1행: "panel.html(UI 정본)에서 verbatim, 디자인 변경 금지".

## 0. 범위 / 비범위
- **범위**: `ui-shared`(화면 10종·훅·SSE 클라·i18n·디자인 토큰) + `web`(Vite SPA 셸) + `landing`(Astro 설득 콘텐츠 FR-954).
- **비범위**: 데스크톱 셸(C4) · 마인드맵·refine-primer(C5) · 광고 실집행(C5, 슬롯 자리만).

## 1. 핵심 설계 결정
### 1-1. 디자인은 verbatim 계승, 셸만 반응형 승격 (사용자 확정)
- `theme.css`(OKLCH 라이트/다크 토큰, 노트 밑줄 브랜딩, 오로라, 칩 부유, `.live` 그라데이션, reduced-motion)를 **한 글자도 바꾸지 않고** `ui-shared/styles/tokens.css`로 이식. 컴포넌트 클래스(.composer·.opt·.chip·.btn 등) 전부 그대로.
- **유일한 변경 = 셸**: v1 `#app{max-width:480px}` 고정폭 → 반응형 2열.
  - 데스크톱(>=1024px): 좌측 사이드바(세션·프로젝트 상시 노출, v1에선 드로어에 숨어 있던 것) + 메인 열(본문은 기존 폭감 유지해 가독성 보존).
  - 태블릿/모바일(<1024px): 사이드바를 드로어로 접고 v1과 동일한 단일 열.
- 근거: 서버 정본·크로스플랫폼 연속성(Q-07)의 가치가 화면에 드러나고, C5 마인드맵 자리도 확보된다. 디자인 언어 이탈 위험 0.

### 1-2. 거대 리듀서 분해 (코드규약 §3 강제)
- v1은 App.tsx 단일 useReducer + State 약 50필드. v2는 **화면별 리듀서로 분리**하고 교차 상태(세션·티어·로케일·한도)만 앱 수준 컨텍스트로 올린다. 단일 거대 리듀서 금지.

### 1-3. 플랫폼 API는 주입 (P3 UI 공유)
- ui-shared는 브라우저/Tauri 분기를 모른다. 파일 첨부·알림·저장소 같은 플랫폼 기능은 **포트로 주입**받아 web(브라우저 구현)과 desktop(C4 tauri 구현)이 각각 제공.

### 1-4. 프롬프트 자산 경계 (SoT §8)
- ui-shared·web·landing 번들에 프롬프트 문자열 금지. portability-guard의 PROMPT_ROOTS에 세 패키지를 등록해 게이트로 강제한다.
