### 03-06. 실측 구조적 취약점 4건 (x-tier 신뢰·자기신고 UUID·클라 state 집계·fail-open)
- 유형: 교훈·경위
- 출처: 배경어휘-사이드탭-결제인증-설계.md §1 (21~25행)
- 폼팩터: 크롬확장-종속(v2에서 재설계 필요) — 코드 파일 경로는 v1 종속이나 취약 패턴(서버 미검증 tier·자기신고 식별자·클라 집계·fail-open)은 v2에서도 반드시 재확인할 일반 교훈
- 원문: > **렌즈가 실제 코드에서 확인한 구조적 취약점:**
  >   - `workers/src/index.ts`의 `readTier()`가 요청 헤더 `x-tier`가 `paid`면 그대로 신뢰(서버에 엔타이틀먼트 SoT 없음). DevTools에서 헤더만 바꾸면 pro 기능 즉시 개방.
  >   - `/summarize`·`/detail`은 tier만 보고 그 tier 출처를 보증하지 않음. `x-user-id`도 자기신고라 매 요청 새 UUID로 보내면 주간 한도 무한 우회(코드 주석이 "Tier3 전까지 구조적 잔존"으로 자인).
  >   - `detailLimitFree`(무료 상세 3회)는 클라 state로만 집계 → 새로고침·직접호출로 무력화.
  >   - 전역 일일 캡·레이트리밋이 `fail-open`(Upstash 장애 시 통과)이라 유일 backstop이 단일 외부 의존.
