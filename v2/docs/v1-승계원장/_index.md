# v1 승계원장 인덱스

v1(크롬 확장) 기획·설계를 v2(웹사이트 + 데스크톱) 리빌딩의 입력으로 무손실 항목화한 인벤토리. **정본은 언제나 `v1/` 원본 파일**이며, 원장은 발췌·항목화본이다. 생성 2026-07-20(47 에이전트 추출·검증 + 3렌즈 교차검증), 등급 분류는 같은 날 사용자 확정.

## 0. 등급 체계 (사용자 확정, 2026-07-20)

폴더 구조가 곧 등급이다.

- **루트 = 정본 승계**: v2 SRS의 직접 입력. 여기 있는 것만 요구사항으로 발췌한다.
- **`_참고/`**: 직접 입력 금지, 필요 시 열람. ★핵심 사유: **결제·수익 모델은 여러 제한이 있는 사이드탭(크롬 확장) 전제 위의 설계라 사이트+프로그램인 v2의 정본이 아니다** — v2 결제·수익 모델은 Phase C에서 웹+데스크톱 전제로 재설계한다. 단 폼팩터 무관 자산(한국법 고지 의무, D1 SoT 원리, webhook 멱등·dunning 필요성, Stripe 한국 불가 같은 시장 사실, LLM·검색 원가 단가)은 참고 가치 유지.
- **`_제외/`**: v2 입력 아님(v1 구현·운영 전용이거나 후속 문서로 전부 대체됨). 삭제하지 않고 격리 보존 — **사이트·프로그램 반응이 좋으면 사이드탭 재개발을 고려할 수 있어** 그때 재활용.

## 2. 결정 최신화 색인 (충돌 쌍 해소)

원장은 시점이 다른 문서를 나란히 담고 있어 **같은 주제의 폐기값과 최신 확정값이 병존**한다. 아래 표의 "v1 최종"이 v1 기준 최신이며, v2 승계 여부는 오른쪽 비고를 따른다.

| 축 | 폐기·구버전 (위치) | v1 최종 (위치) | v2 비고 |
|---|---|---|---|
| 구현 형태 | 1차=독립 웹앱(01a A-2) → 크롬 확장 전환(01c §11) | 크롬 확장 | **v2 = 웹사이트 + 데스크톱(Win/Linux), 2026-07-20 사용자 결정.** v1의 "웹앱→확장" 전환 사유(01c·_제외/04)는 재검토 참고 |
| 결제·수익 모델 | — | 가격 ₩3,900/$2.99, 포트원 V2(토스+엑심베이), PAYMENT_ENABLED b안 (_참고/03·02b/c) | **v1 최종이어도 v2 정본 아님** — 확장 제약 위 설계라 Phase C에서 재설계(사용자 확정). 한국법 고지·webhook 멱등·dunning 필요성 등 원리만 참고 |

## 3. 사용 주의

- **폼팩터 태그는 참고용(advisory)** — 항목 단위 태그라 원문이 혼합 성격일 수 있다(예: _참고/03-56 "무관" 항목 안에 CWS 조항). 원문을 읽고 판단.
- **유형=확정결정은 "그 시점의 확정"** — 번복 여부는 §2 표와 항목 내 정정 주석으로 확인.
- 인접 청크(a/b/c) 경계엔 완결용 소량 중복이 있다(각 파일 헤더에 명시).
- 스코프: v1/_archive/(과거 산출물)와 v1/UI_UX/는 원장 미포함 — 필요 시 별도 라운드.
- 검증 이력: 청크별 원본 재대조(누락 보충 11건) + 3렌즈(coverage→꼬리 4청크 보완, fidelity→태그 정정 2건, usability→§2 색인 신설).
