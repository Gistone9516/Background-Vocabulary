## 2. 결정 최신화 색인 (충돌 쌍 해소)

원장은 시점이 다른 문서를 나란히 담고 있어 **같은 주제의 폐기값과 최신 확정값이 병존**한다. v2 명세 작성 시 아래 표의 "최신 확정"만 승계 대상이고, 폐기값은 경위 참고용이다.

| 축 | 폐기·구버전 (위치) | 최신 확정 (위치) | v2 비고 |
|---|---|---|---|
| 구현 형태 | 1차=독립 웹앱(01a A-2) → 크롬 확장 전환(01c §11, 2026-06 확정) | **v2 = 웹사이트 + 데스크톱(Win/Linux), 2026-07-20 사용자 결정** | v1의 "웹앱→확장" 전환 사유(01c·04)를 v2 설계 때 재검토할 것 |

## 3. 사용 주의

- **폼팩터 태그는 참고용(advisory)** — 항목 단위 태그라 원문이 혼합 성격(예: 03-56 "무관" 항목 안에 CWS 심사 등 확장 종속 조항, 03-45 "종속" 항목 안에 대부분 유효한 준비물)일 수 있다. 필터링 근거로 기계적으로 쓰지 말고 원문을 읽고 판단.
- **유형=확정결정은 "그 시점의 확정"** — 이후 번복 여부는 §2 표와 각 항목의 정정 주석으로 확인(09-39·12-1에는 검증 정정이 주석으로 박혀 있음).
- 인접 청크(a/b/c) 경계엔 완결을 위한 소량 중복 수록이 있다(각 파일 헤더·출처에 명시).
- 스코프: v1/docs 9종 + v1 README 2종 + REFACTOR-NEXT + 제품근거 메모리. **v1/_archive/(과거 산출물, gitignore)와 v1/UI_UX/는 미포함** — 필요 시 별도 라운드.
- 검증 이력: 청크별 원본 재대조(누락 보충 11건, 요약화 0건 유지) + 3렌즈(coverage가 꼬리 4청크 부재 적발 → 01c·02c·11c·12d로 보완, fidelity가 태그 오류 2건 적발 → 09-39·12-1 정정, usability가 충돌 쌍 병존 적발 → 본 §2 색인 신설).
