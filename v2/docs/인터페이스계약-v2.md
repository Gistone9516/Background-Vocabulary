# 배경노트 v2 — 인터페이스 계약 (SoT v2.0 초안, 2026-07-21)

> buildflow ②단계 SoT. **충돌 시 이 문서가 권위(SoT wins).** 패키지별 스펙은 여기에 conform.
> 상위 정본: 요구사항 = `SRS-요구사항명세.md`(v1.0) · 기술 결정 = `ADR-아키텍처결정.md`(v1.0) · v1 계약 = `v1/docs/배경어휘-사이드탭-인터페이스계약.md`(출발점, 본 문서가 대체).
> 표기: [승계] = v1 계약 그대로 유효, [개정] = v1에서 변경, [신규] = v2 신설.

## 0. 권위 규칙 [승계·개정]
- 화면 흐름·노출 필드·문구·태깅 UX = 프론트(ui-shared) 설계가 권위. 백엔드가 굽힌다.
- 로케일 라우팅·RAG 필수·고위험 거부·TTL·모델 라우팅·티어 판정·한도 = 백엔드가 권위. 프론트가 따른다.
- [개정] 영속 데이터(세션·어휘 자산·지식 상태·프로젝트)의 정본 = **서버 DB**(SRS Q-07). 클라이언트 저장소는 캐시일 뿐이며 충돌 시 서버가 이긴다.

## 0-1. ★ 이식성 불변 경계 [승계 — AWS가 "나중"에서 "지금"이 됐어도 경계는 동일]
- **core/·shared/·ui-shared/ 안 런타임 전역 0건.** AWS SDK·Tauri API·Node 전용 API·DOM 밖 브라우저 API 직접 호출 금지. 외부 자원은 전부 포트(`LlmClient·SearchProvider·CacheStore·UserRepository·AssetRepository·SessionRepository·UsageCounter·EnvConfig`) 뒤로 추상화하고 주입(DI).
- **웹표준만 사용:** fetch · Request/Response/Headers · Web Streams · Web Crypto · TextEncoder/Decoder · URL · AbortController/AbortSignal.
- **런타임·공급자 특수성은 adapters/에만:** Lambda 부트(streamHandle)·Data API 클라이언트·Upstash REST·Tavily REST·Tauri invoke·로컬 Node 부트(@hono/node-server)·로컬 DB 드라이버.
- **하드플로어 가드 3중(NFR-505, ADR-004):** ① TS 프로젝트 레퍼런스(참조 없는 패키지는 컴파일 불가) ② portability-guard(정규식, v1 승계) ③ dependency-cruiser(순환·역참조). 셋 중 하나라도 위반 = 빌드 실패.

## 0-2. ★ 실행 3계층 [신규 — ADR-003 로컬 dev 모드]
| 계층 | 부트 | DB | 캐시·리밋 | LLM·검색 | 용도 |
|---|---|---|---|---|---|
| mock | 없음(클라 목) | 없음 | 없음 | 목 | UI 개발 |
| local | `adapters/local`: @hono/node-server | 로컬 PostgreSQL(Docker) 기본, SQLite 대안 | 인메모리 또는 Upstash | .env 실키 또는 목 | 통합 테스트 |
| aws | `adapters/aws`: Lambda Function URL(RESPONSE_STREAM) + streamHandle | Aurora Serverless(PG) + Data API | Upstash REST | Secrets 실키 | staging·prod |
- **같은 Hono 앱 인스턴스**를 세 부트가 공유한다. 앱 조립(라우트 등록)은 `adapters/http-app`(가칭)에 있고 부트만 계층별. 라우트 코드에 계층 분기 금지 — 분기는 주입된 포트 구현이 담당.
- SQL은 PostgreSQL 방언을 정본으로 작성. SQLite 대안 어댑터는 방언 차이를 어댑터 내부에서 흡수하되, 통합 테스트 통과 기준은 PG다.

## 2. 공유 타입 (shared/types.ts) — v1 승계 + v2 델타

### 2-1. 파이프라인 타입 [승계]
`Choice`(label, domain_tags — **deeper 필드 없음**, SRS §6.1) · `Term`(필수 term/kind/group/priority/why/one_line/tag, 조건부 direction/use_example/context_note/relates_to/order/gap_type) · `Source{title,site,url}` · `Prompt1~5 In/Out` · `RecommendInput`(domain_risk, exclude) · `StreamEvent` · enum `JobType` 9종·`GapType` a~e·`Tag`(알아/몰라/적용모름) — 전부 v1 계약 §2 그대로. kind는 자유 문자열.

### 2-2. 인증·엔타이틀먼트 타입 [승계 §8-1]
`SubscriptionStatus` 8종 · `UserRecord` · `Entitlement`(effective_tier = paid OR grace) · `AccessTokenClaims`(15분, epoch 초) · `RefreshTokenClaims`(30일, jti 블랙리스트) · `NewUser` · `EntitlementPatch`(occurred_at 순서 가드) — v1 계약 §8-1/8-2 그대로. 시각 단위 경계 유지: JWT=초, DB·도메인=밀리초, 변환은 adapters.

### 2-3. 영속 도메인 타입 [신규 — 서버 정본화]
```ts
export interface SessionRec {            // 탐색 세션(진행 중 또는 완료)
  session_id: string; user_id: string;
  topic: string; area: string | null; domain_risk: DomainRisk;
  job_type: JobType[]; gap_type: GapType[] | null;
  user_condition: string | null; context_object: string | null;
  narrow: NarrowSnap | null;             // null이면 생성 완료(불변식: narrow 존재 ⟺ 생성 미완, v1 승계)
  generated: Term[] | null;              // 생성된 리스트 전체(담기 0개여도 보존, FR-702)
  project_id: string | null; pinned: boolean;
  created_at: number; updated_at: number;
}
export interface NarrowSnap {            // 좁히기 진행 스냅샷(매 턴 저장, FR-701)
  question: string; choices: Choice[];
  answers: { label: string; action: "선택" | "더깊이제외" | "어려워요" }[];
  turns_left: number;                    // 재개 시 (현재 plan, answers)로 재계산이 정본(v1 교훈)

  domain_tags: string[];                 // 마인드맵 크로스 도메인 엣지 재료(FR-312)
  project_id: string | null; created_at: number;

export interface KnowledgeState {        // 지식 상태(FR-502). unconfirmed는 클라 전용이라 없음

export interface Project { project_id: string; user_id: string; name: string; created_at: number }
export interface PrimerDoc {             // 구조화 프라이머(FR-604) — 직렬화 정본
  area: string; task_intent: string;
  user_condition?: string; context_note?: string;
  known_terms: string[]; unknown_terms: string[];
  refined?: { audience?: string; goal_detail?: string; constraints?: string[] };  // FR-607 산출
}
export interface MindMap {               // GET /map 응답(FR-312)
  scope: "all" | "project"; project_id?: string;
  nodes: { id: string; term: string; kind: string; group: string; session_id: string }[];
  edges: { from: string; to: string; kind: "group" | "relates_to" | "cross_domain" }[];
}
```

### 3-2. 인증 [승계 §8-4 개정 — 폼팩터 재설계(NFR-307)]
| 엔드포인트 | In | Out | 비고 |
|---|---|---|---|
| POST /auth/google | { code, code_verifier, redirect_uri } | { access, refresh, user } | 표준 OAuth Authorization Code + PKCE. id_token iss/aud 검증, client_secret 서버 전용 |
| POST /auth/refresh | { refresh } | { access, refresh } | jti 블랙리스트 검사(refresh 시점만, networkless access 검증 유지) |
| POST /auth/logout | { refresh } | 204 | jti 블랙리스트 등록 |
| GET /subscription/status | (Bearer) | Entitlement | [승계] |
| POST /checkout | (Bearer) | CheckoutSession 또는 coming_soon | [승계] PAYMENT_ENABLED b안 유지(TR-04) |
- 웹: SPA → OAuth redirect(자기 오리진 콜백). 데스크톱(Tauri): 시스템 브라우저 + 루프백/커스텀 스킴 콜백 — 구현은 adapters/tauri, 코어 계약은 동일(code+PKCE).
- launchWebAuthFlow·chrome.identity 코드는 이식 금지(v1 잔재).

### 3-3. 영속 CRUD [신규 — 서버 정본. 전부 로그인 필수(Bearer), 자기 소유 행만]
| 엔드포인트 | 동작 |
|---|---|
| GET /sessions?scope=… | 세션 목록(검색 q, project_id, pinned 필터. 커서 페이지네이션) |
| GET /sessions/:id | 단건(narrow·generated 포함) |
| PUT /sessions/:id | 진행 스냅샷 upsert(매 턴 저장, 멱등 — 클라 생성 session_id=UUIDv7) |
| DELETE /sessions/:id | 소프트 삭제(실행취소 유예 후 하드 삭제) |
| PUT /sessions/:id/keep | 담기: AssetTerm 생성/해제 |
| GET /assets?project_id=… | 어휘 자산 목록 |
| GET/POST/DELETE /projects | 프로젝트 CRUD(삭제 시 세션 소속만 해제, 세션 보존 — v1 규칙 승계) |
| PUT /knowledge | KnowledgeState 일괄 upsert(태깅 변경 배치) |
- **동기화 규칙(FR-503·FR-902):** 정본=서버. 클라(웹 IndexedDB/localStorage, 데스크톱 로컬 캐시)는 read-through 캐시 + 오프라인 열람용. 쓰기는 online-only(오프라인 쓰기 큐는 범위 밖 — 생성 기능 자체가 온라인 전용이므로 충돌 표면 최소). 충돌 시 updated_at 큰 쪽이 아니라 **서버 수리 원칙**: 클라는 서버 응답으로 자기 캐시를 덮는다.
- **미로그인 사용자:** 파이프라인은 익명 허용(free 한도, NFR-301). 영속 CRUD는 로그인 필수 — 미로그인 세션은 클라 로컬에만 임시 보관하고 로그인 시 서버로 승격(1회 마이그레이션 업로드).

## 4. 게이팅·한도 의사코드 [승계 개정]
```
resolveTier(req):  Bearer JWT 검증(HS256, exp) → claims.tier   // networkless, x-tier류 헤더 완전 무시(C-21)
                   DEV_MODE && DEV_FORCE_TIER → override(로컬 전용)
gate(/classify):   globalDailyCap(전역) → ipRateLimit → freeWeekly(user_id 또는 익명 키) 차감 → 통과
gate(/detail):     free면 서버 카운터 증가·상한 검사(NFR-304)
gate(/summarize, /refine-primer): tier != paid → 402
사용량 저장:        Upstash(레이트리밋·주간 카운터) — 장애 시 fail-open(NFR-404)
```
- 익명 키 전략: IP+day 기반(NFR-301). 계정 기반 정밀 카운트는 로그인 사용자에 적용.

## 5. 스트리밍 프로토콜 [승계]
- /recommend: `text/event-stream`. 이벤트 = `term`(Term 1건) · `meta`(terms_rationale 등) · `done` · `error`(정화된 메시지). Lambda 어댑터는 streamHandle로 동일 이벤트 직렬화(adapters/aws), 로컬은 node-server SSE. **클라 AbortController → fetch signal → 파이프라인 → LlmClient.streamTerms까지 전파**(FR-310).
- 이벤트 스키마·순서는 mock(클라 목)도 동일하게 재현한다(3계층 등가성).
## 6. 데이터베이스 스키마 (PostgreSQL 정본) [신규 — v1 D1 0001_init 승계 확장]

```sql
-- users: v1 D1 K1 스키마 승계(컬럼 동일 취지), PG 타입으로. PK=내부 user_id.
CREATE TABLE users (
  user_id        TEXT PRIMARY KEY,           -- UUIDv7
  email          TEXT NOT NULL UNIQUE,
  google_sub     TEXT UNIQUE,
  tier           TEXT NOT NULL DEFAULT 'free',
  subscription_status TEXT NOT NULL DEFAULT 'none',
  expires_at BIGINT, current_period_end BIGINT,
  cancel_at_period_end BOOLEAN NOT NULL DEFAULT FALSE,
  grace_until BIGINT, failed_payment_count INT NOT NULL DEFAULT 0,
  next_retry_at BIGINT, last_failure_code TEXT,
  current_price INT, currency TEXT, billing_interval TEXT NOT NULL DEFAULT 'monthly',
  pg_provider TEXT, created_at BIGINT NOT NULL
);
CREATE TABLE sessions (
  session_id TEXT PRIMARY KEY, user_id TEXT NOT NULL REFERENCES users(user_id),
  topic TEXT NOT NULL, area TEXT, domain_risk TEXT NOT NULL,
  job_type JSONB NOT NULL, gap_type JSONB,
  user_condition TEXT, context_object TEXT,
  narrow JSONB,                                -- NarrowSnap. NULL ⟺ 생성 완료
  generated JSONB,                             -- Term[]
  project_id TEXT, pinned BOOLEAN NOT NULL DEFAULT FALSE,
  deleted_at BIGINT,                           -- 소프트 삭제(실행취소)
  created_at BIGINT NOT NULL, updated_at BIGINT NOT NULL
);
CREATE INDEX idx_sessions_user ON sessions(user_id, updated_at DESC) WHERE deleted_at IS NULL;
CREATE TABLE assets (
  asset_id TEXT PRIMARY KEY, user_id TEXT NOT NULL REFERENCES users(user_id),
  session_id TEXT NOT NULL, term JSONB NOT NULL,
  term_norm TEXT NOT NULL,                     -- 정규화 키(중복 담기 방지·맵 노드 키)
  domain_tags JSONB NOT NULL, project_id TEXT, created_at BIGINT NOT NULL,
  UNIQUE(user_id, session_id, term_norm)
);
CREATE INDEX idx_assets_user_project ON assets(user_id, project_id);
CREATE TABLE knowledge (
  user_id TEXT NOT NULL REFERENCES users(user_id),
  term_norm TEXT NOT NULL, tag TEXT NOT NULL, updated_at BIGINT NOT NULL,
  PRIMARY KEY(user_id, term_norm)
);
CREATE TABLE projects (
  project_id TEXT PRIMARY KEY, user_id TEXT NOT NULL REFERENCES users(user_id),
  name TEXT NOT NULL, created_at BIGINT NOT NULL
);
-- 결제 이벤트·billing_locks·notification_log·consents: v1 K1 승계(Phase 1 활성화, 컬럼 정의는 유지)
```
- JSONB 채택 근거: narrow/generated/term은 클라 정본 구조의 스냅샷이라 정규화 이득이 작고 스키마 진화가 잦다. 조회 축(사용자·프로젝트·시간)만 컬럼·인덱스로.
- 마이그레이션: `adapters/*/migrations/` 순번 SQL. 로컬(Docker PG)과 Aurora가 같은 파일을 사용.

## 8. 프롬프트 자산 경계 [신규 — NFR-311]
- 프롬프트 빌더 본문(buildPrompt1~6·preview·relate)은 **core 패키지에만** 존재한다. shared에는 시그니처(입출력 타입)만.
- ui-shared·web·desktop·landing 번들에 프롬프트 문자열이 포함되면 안 된다 — portability-guard에 프롬프트 토큰 검사 추가(경계 게이트의 일부).
- API 에러 응답은 정화 계층을 거쳐 프롬프트·내부 경로를 절대 노출하지 않는다(v1 에러 정화 승계).
- 시스템 프롬프트 유출 유도 입력 방어 지시를 P1~P6 공통 가드 블록으로 둔다.
